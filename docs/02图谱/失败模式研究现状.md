# LLM OPD 失败模式研究现状与开放问题

更新时间：2026-06-19

## 阅读口径

本文把 OPD 相关 failure mode 分成三类：

- **已被后续论文直接研究并尝试解决**：后续论文明确把某类失败现象作为动机，并改动 OPD 的轨迹、teacher signal、KL/estimator、token credit 或训练 pipeline。
- **已有局部解决，但还没闭环**：已有诊断或 mitigation，但仍依赖强假设、任务较窄、proxy 不稳、成本较高，或只解决一个子场景。
- **仍待研究**：目前主要以 limitation、future work、综述空白或新论文动机出现，还没有成熟的算法闭环。

覆盖范围：

- 早期 26 篇：已经做过逐篇 failure / mechanism / limitation 精读。
- 2026-05+ 增量 59 篇：来自 `LLM_OPD论文筛选总结.md` 的增量保留表；本轮已用 `pypdf` 从本地 PDF 全文抽取，并重点核对 `Limitations`、`Future Work`、`Discussion`、`Conclusion` 等段落。增量论文仍按主线聚合，不在本文逐篇复写所有原文。

## 总体结论

早期 OPD 研究主要围绕一个核心问题展开：

```text
off-policy KD 有 exposure bias
  -> OPD 改到 student-generated states 上训练
  -> 但 teacher signal、KL 方向、token credit、privileged context 又引入新的失败模式
```

2026-05+ 增量之后，OPD failure mode 的研究重心明显前移：不再只问“OPD 是否比 SFT/RL 好”，而是更细地问：

```text
在哪些 student states 上监督？
teacher signal 什么时候可靠？
哪些 token / step 真可教？
sampled-token KL 的估计是否稳定？
privileged context 会不会泄漏或与已内化参数冲突？
OPD 和 RLVR 应该如何分工？
black-box / rubric / safety 等语义信号能否替代 logits？
```

综合当前文档中的 85 篇核心条目，failure mode 可以归为 16 条主线：

1. off-policy / static KD 的 exposure bias 与 state distribution mismatch；
2. teacher-student gap 大时 FKL / RKL / strict imitation 的散度方向失败；
3. 标准 OPD 把 reward term 与 KL regularization 绑定，导致 teacher boundary / capability ceiling；
4. teacher-student thinking-pattern mismatch、teacher novelty 不足与 OPD success condition 不清；
5. privileged-information self-distillation 的信息不对称、leakage、KL stagnation；
6. teacher exposure 过强、privileged residual 不可迁移、context internalization 后 context-return 退化；
7. RLVR sparse outcome reward 的 token-level credit assignment 不足；
8. 普通 OPD 对 correct / incorrect rollouts 均匀施加 teacher supervision，忽略 signal quality；
9. OPSD 在长数学推理中更像 compaction，而不是 failed-trajectory correction；
10. 长轨迹中的 prefix drift、supervision fidelity decay、off-policy teacher decay 和 KL agreement trap；
11. sampled-token / advantage-style OPD 的高方差、heavy-tail、zero-advantage 和 stop-gradient bias；
12. token-level supervision 冗余、错误或不可教，缺少 token teachability / path-conditioned credit；
13. teacher signal 本身不可靠，有 single-teacher capability ceiling、confidence miscalibration 或 teacher rollout 随机性；
14. offline / asynchronous / private OPD 的 teacher consistency、freshness、cache drift 和 privacy threat model；
15. black-box teacher 无 logits，white-box KL OPD 不可用，需要 response、rubric、verification、constitution 等语义信号；
16. safety、subjective / creative、long-horizon agentic、cross-tokenizer / cross-family 等外部约束还未形成统一评测和理论。

其中，第 1-12 类已经有较密集的后续论文接续；第 13-16 类已有局部方案，但仍明显不闭环。

## 后续论文接续关系

### 1. 从 off-policy mismatch 到 state-distribution view

早期 KD / SeqKD 的核心 failure 是 **exposure bias**：student 训练时看 teacher 或固定数据的 prefixes，推理时却只能沿自己的 prefixes 继续生成。OPD 的第一层价值是把 supervision 从 teacher states 移到 student states。

新的进展是：`2605.22731` 把 SFT、RL、OPD 的差别进一步抽象为 **state distribution** 差别，而不是只看 MLE、KL、policy gradient 这些 loss 名称。`2605.16826` 则把 prefix source 和 KL direction 解耦，说明“在哪里监督”和“用什么散度监督”是两个独立轴。

仍待研究：multi-turn agent 的 state 不只是文本 prefix，还包括工具结果、环境状态、历史失败、memory；如何覆盖 informative states 仍未解决。

### 2. 从 KL 方向失败到 estimator / trust-region 稳定化

早期主线是 FKL / RKL 方向选择：Veto、EOPD、HPD 都在处理 mode-covering 与 mode-seeking 的取舍。

新增论文把问题推进到 **sampled-token estimator** 和 **更新幅度控制**：

- `2605.07865` vOPD：用 control variate baseline 降低 Monte Carlo KL 方差。
- `2606.17199` PowerOPD：把 sampled-token RKL 的高方差、heavy-tail、早期高梯度病理作为核心 failure。
- `2606.01039` OPD+：指出 stop-gradient advantage 可能造成目标和梯度不一致。
- `2606.01249`、`2605.31159`：用 trust region / behavior blending 限制大 policy gap 下的坏更新。
- `2606.03532`：把 self-OPD 稳定性和 teacher update schedule / isolation period 联系起来。

仍待研究：full-vocab KL、top-k/subset KL、sampled-token estimator、explicit KL loss、token-level reward、token-level reweighting 的等价条件和偏差边界。

### 3. 从 strict imitation 到 OPD success predictor

`2604.13016` 说明 OPD 不一定因为 teacher 强就有效，核心条件包括 thinking-pattern compatibility 和 teacher novelty。`2605.10889`、`2605.11182`、`2605.11739`、`2605.30070`、`2606.07082` 进一步把问题变成：

```text
训练前或训练早期，能否预测这个 teacher-student-data 组合是否值得做 OPD？
```

可能的 predictor 维度包括：

- teacher-student top-k overlap / state compatibility；
- teacher entropy / fidelity / discriminativeness；
- initial student-self-teacher gap；
- teacher novelty；
- early KL dynamics；
- parameter-space update geometry。

目前这些指标分散在不同论文中，还没有统一成可执行的 OPD go/no-go detector。

### 4. 长轨迹 failure 成为新增主线

2026-05+ 增量中最明显的新主线是长轨迹：

- `2605.07804` Prune-OPD：prefix drift 后 teacher supervision 不再 local exploitable。
- `2605.30833` SFD：teacher 的 supervision fidelity 随 student prefix 变长而下降。
- `2605.27028` / `2605.31490`：full rollout 不一定必要，后段 token 可能只增加噪声和成本。
- `2606.09471`：坏 prefix 上低 KL agreement 可能是 trap，而不是有效纠错。
- `2606.00305`：token-local KL 无法充分 reshape near-future trajectory。
- `2605.19433` / `2606.08432`：需要 backtracking 或 trajectory refinement。

这说明 OPD 的基本单位正在从 token 扩展到 prefix、span、step、window 和 trajectory。

### 5. 从 privileged self-distillation 到 context governance

`2604.03128` 已经把 privileged OPD 的核心问题拆成 mutual-information gap 和 gradient-direction contamination。新增论文进一步说明：只说“用 PI”太粗，关键是 **PI 如何暴露、如何残差化、如何重新进入推理上下文**。

代表进展：

- `2605.11458`：teacher exposure 不是越多越好，要和 student 当前能力匹配。
- `2606.10385`：privileged OPD 不应 absolute imitation，而应只蒸馏 anchored residual。
- `2606.11627`：context 被内化后，重新给回 context 可能让 student 退化。
- `2606.19327`：rubric 可以替代单一 reference trajectory，作为更结构化的 privileged supervision。

开放问题：PI leakage benchmark 需要从“是否复述答案”扩展到“是否学到不可见状态依赖的行为痕迹”。

### 6. 从 RLVR sparse reward 到 dense token / step credit

RLVR 的 outcome reward 稀疏问题早期由 SDPO、SD-Zero、SCOPE 承接。新增论文把 credit assignment 做得更细：

- `2606.00172` CAST：处理 GRPO 全对/全错 group 中 zero-advantage 的无梯度问题。
- `2606.18810` Self-Conditioned Credit：用 own solutions 做 token credit。
- `2605.12652` Multi-Rollout OPD：利用同 prompt 的 peer success/failure。
- `2605.17497` SSOPD：正确 completion 是 witness，错误 completion 提供 correction prefix。
- `2606.15576` Path-Conditioned SD：只在 path divergence 处 localize credit。

核心开放问题不是“有没有 dense signal”，而是 dense signal 是否是 **causal credit**。

### 7. 从 full-token OPD 到 selective / teachability-aware OPD

TIP 已经说明 full-token OPD 成本高且冗余。新增论文把 token selection 从 entropy/divergence 推进到 teachability 和 causality：

- TRACE：只在 privileged annotator 标出的 critical spans 上蒸馏，减少 leakage 和无效梯度。
- Token Teachability：不是所有 disagreement 都 learnable，必须判断当前 student 是否能吸收。
- Position-Weighted OPSD：teacher token reliability 随位置变化。
- RLCSD：privileged gap 可能落在 style tokens，要用 contrastive signal 抵消。
- Input-Specific Credit / PMI：区分 generic correlation 和 input-specific credit。

开放问题：如何把 semantic role、proof step、tool-call field、error type 和 process reward 纳入 token selection。

### 8. 从 white-box logits 到 semantic OPD

传统 white-box OPD 依赖 teacher logits。现实中很多 teacher 只能给 response、rubric、verbal feedback、verification 或 constitution。

已有路线：

- SODA：teacher response vs static student negative。
- ROPD：prompt-specific rubric + verifier。
- OmniOPD：logit-free，使用 speculative verification 和 Monte Carlo semantic estimation。
- RCSD：rubric-conditioned teacher，将 rubrics 当 privileged teacher-side supervision。
- Constitutional OPD：把 constitution 当安全场景的 teacher context。

开放问题：这些语义信号能否恢复 teacher distribution 的 uncertainty、多模态 alternative modes 和 token-level local preference，目前还不清楚。

### 9. 从 OPD/RLVR 对立到 sparse-to-dense pipeline

`2605.12483` 的 sparse-to-dense principle 把 OPD 和 RLVR 的关系讲得更清楚：

```text
RLVR / GRPO:
  更适合探索和发现新行为

OPD / OPSD:
  更适合压缩、校准、局部 credit 和把 context / feedback / teacher signal 内化
```

这和 `2605.06188` 的 conclusion 一致：OPSD 更像 post-RL compaction，而不是直接替代 RLVR 的 correction mechanism。

开放问题：什么时候从 RLVR 切到 OPD，什么时候从 OPD 切回 RL 或 verifier calibration，目前还没有统一调度器。

## 论文自述局限与未来方向

这一节只总结论文原文中明确出现或在 discussion / conclusion 中直接指向的不足和可改进方向，不把摘要中的动机重复算作 limitation。

| 主线 | 论文自述的主要局限 | 作者或论文指向的改进方向 | 涉及论文 |
|---|---|---|---|
| 规模和领域外推 | 很多结论仍在 Qwen / Olmo 等少数模型家族、1.5B-8B 或 30B 以内模型、数学 reasoning 上验证；大模型、长训练、full-parameter update、非 Qwen backbones、多模态、代码、开放式对话、tool-use 的外推仍不足。 | 扩大模型规模和家族；加入 code、instruction following、open-ended generation、multi-turn agents、multimodal reasoning；做多 seed 和更长 horizon 验证。 | `2605.07865`, `2605.11182`, `2605.11458`, `2605.16826`, `2605.22731`, `2605.26844`, `2605.27028`, `2606.00172`, `2606.03532`, `2606.07082`, `2606.09471`, `2606.17199`, `2606.18810` |
| OPD 适用条件和预测器 | 许多论文承认没有 universal recipe：teacher 更强不一定更好，OPD 是否有效取决于 teacher-student compatibility、context design、student state、initial gap、teacher novelty 和任务结构。 | 训练前或训练早期预测 OPD 是否值得做；用 rollout alignment、student-teacher divergence、initial self-teacher gap、early geometry、teacher novelty 做 go/no-go 或 routing 信号。 | `2604.13016`, `2605.10889`, `2605.11182`, `2605.11739`, `2605.30070`, `2606.07082` |
| 长轨迹和监督 horizon | Prune-OPD、ESR、ADWIN、KAT、TRD 等都指出 full rollout 不是总有用，但现有 horizon 控制多为固定、batch-level 或启发式；某些任务的关键信号可能在 suffix、final answer、proof closure、code executability 或长程依赖处。 | 从固定截断走向 per-sample / prefix-aware / step-aware horizon controller；用 KL、entropy、teacher confidence、log-ratio variance、near-future drift、gradient alignment 等共同决定截断、回退或 refine。 | `2605.07804`, `2605.19433`, `2605.27028`, `2605.28396`, `2605.31490`, `2606.00305`, `2606.08432`, `2606.09471` |
| teacher signal reliability | 现有方法常假设 teacher ranking、confidence、top-k overlap、entropy 或 KL agreement 能代表可靠性，但论文也承认这些 proxy 有范围限制：低 KL 可能是 trap，高 entropy 可能是良性多解，top-k overlap 只是轻量近似。 | 建立统一 teacher-signal reliability estimator；把 confidence、discriminativeness、teachability、branch viability、path divergence 和 outcome feedback 联合起来。 | `2603.07079`, `2605.07804`, `2605.09725`, `2605.21606`, `2605.26844`, `2605.30833`, `2606.09471`, `2606.10385` |
| token / step credit | 多篇论文承认 trajectory-level correctness 与 token-level KL 不必因果一致；同一 prompt 的 peer success/failure、path divergence、reflection、rubric、process feedback 尚未形成统一 credit 机制。 | 从全 token KL 转向 token teachability、position weighting、contrastive credit、path-conditioned credit、Bayesian recursion、process / step certificate；把 semantic role、proof step、tool-call field 纳入选择性监督。 | `2604.10688`, `2604.14084`, `2605.10194`, `2605.11609`, `2605.11613`, `2605.12652`, `2605.17497`, `2605.18529`, `2605.21606`, `2605.21851`, `2605.26844`, `2606.00172`, `2606.15576`, `2606.18810` |
| correction vs compression | OPSD Compresses、ROSD、SSOPD 等都指向同一问题：OPSD 对 correct rollouts 更像压缩和校准，对 incorrect rollouts 不一定能修复，甚至会损害 accuracy 或泛化。 | 明确区分 correction stage 和 compaction stage；让 RLVR / search 负责探索和发现能力，OPD 负责压缩、局部 credit、context internalization；为 failed trajectories 设计 witness、reflection、backtracking 或 trajectory refinement。 | `2605.06188`, `2605.12483`, `2605.17497`, `2605.28014`, `2606.08432`, `2606.15576` |
| sampled-token estimator 和 KL 目标 | vOPD、OPD+、SDPG、PowerOPD 都承认 sampled-token KL、full-vocab KL、path-wise loss、score-function reward 和 token reweighting 并不自动等价；现有实验也缺少系统 scaling 和 divergence 比较。 | 明确不同 estimator 的无偏性、方差、计算成本和真实优化目标；比较 full-vocab、top-k、sampled-token 在不同 divergence 下的 tradeoff；把 sequence-level KL 纳入研究。 | `2605.07865`, `2606.01039`, `2606.04036`, `2606.09304`, `2606.17199` |
| trust region、teacher schedule 和 offline freshness | 大 policy gap、teacher update 太频繁、异步 rollout、offline cache 和 teacher consistency 都可能带来 stale supervision 或坏更新；现有方案常需要 schedule、threshold 或 isolation period。 | 统一 teacher/student drift、cache freshness、teacher refresh、trust region 和 filter/reweight 的控制理论；让 teacher 是否移动由 reward improvement、length-tail safety、student consolidation 等状态决定。 | `2602.22495`, `2604.13010`, `2605.17862`, `2605.31159`, `2606.01249`, `2606.02684`, `2606.03532` |
| privileged context 和 leakage | PI/OPSD 论文持续指出 privileged context 不是越多越好：full teacher exposure、absolute imitation、answer-bearing trace、context reintroduction 都可能造成 leakage、shortcut、hindsight trap 或 context-induced degradation。 | 从“是否使用 PI”转向“如何治理 PI”：per-sample exposure、anchored residual、evidence mask、context-removability objective、context-return robustness、PI 类型 taxonomy。 | `2602.04942`, `2604.03128`, `2605.10194`, `2605.11458`, `2605.23493`, `2606.10385`, `2606.11627` |
| black-box / semantic teacher signal | Rubric、language feedback、speculative verification、constitution 等能绕过 logits，但论文承认这些信号依赖 judge / verifier / rubricator 质量，且未必恢复 teacher uncertainty 或 token-local preference。 | 校准 rubric 和 verifier；用多样本 response、semantic similarity、criterion-level feedback、verbal uncertainty 近似 distribution signal；扩展到 subjective / creative / open-ended 任务。 | `2604.03873`, `2605.07396`, `2605.15113`, `2606.01476`, `2606.03089`, `2606.19327` |
| 安全、隐私和高风险任务 | SafeSteer 依赖 base model 已有拒答能力；UniSD 指出 self-distillation 会继承 factual errors、bias 和 unsafe behavior；DP-OPD 的 threat model 还涉及 metadata、control codes、teacher-query budget。 | 建立 safety / bias / factuality regression tests；研究 localized safety OPD 在更大模型、VLM、DLLM 上的泛化；细化 private corpus、query metadata 和 teacher inference 的隐私边界。 | `2604.04461`, `2605.06597`, `2606.02530`, `2606.03089` |
| 工程成本 | 很多修复会增加 teacher forward、extra rollout、reflection generation、OT 计算、online teacher decoding、multi-rollout context 或 teacher-student co-residency；部分方法的 token budget 不是 wall-clock speedup。 | 研究成本-收益曲线；设计 top-k / chunk-level / sparse-token / early-exit / cache-refresh 版本；在真实 serving、teacher rate-limit 和长上下文设置下评估。 | `2605.07865`, `2605.12652`, `2605.18529`, `2605.19433`, `2605.26844`, `2605.31159`, `2606.00305`, `2606.08432`, `2606.10385` |

## 仍待研究的问题

### A. 机制层面

| 开放问题 | 为什么还没解决 | 相关论文 |
|---|---|---|
| OPD success predictor | 现有指标分散：thinking-pattern overlap、teacher novelty、student exploration、teacher fidelity、initial gap、geometry；还没有统一训练前预测器。 | `2604.13016`, `2605.10889`, `2605.22731`, `2605.30070`, `2606.07082` |
| teacher signal reliability estimator | entropy、PPL、top-k overlap、fidelity、KL agreement、teachability 都是 proxy，适用范围不同。 | `2603.07079`, `2605.07804`, `2605.30833`, `2606.09471`, `2605.26844` |
| correction vs compression detector | OPSD 常压缩正确推理而非修复失败；需要判断一次 OPD update 是 correction、compression 还是 error reinforcement。 | `2605.06188`, `2605.12483`, `2605.17497` |
| token-level causal credit | correctness 是 trajectory-level，teacher KL 是 token-level，但二者不必因果一致。 | `2604.10688`, `2605.10194`, `2605.11613`, `2606.15576`, `2606.18810` |
| privileged information leakage taxonomy | 不同 PI 形式的 leakage 路径不同；context-return 退化说明内化后的 policy 也会和外部 context 冲突。 | `2604.03128`, `2605.11458`, `2606.10385`, `2606.11627` |
| sampled-token estimator 的真实目标 | path-wise KL loss、score-function reward、token reweighting 可能对应不同梯度；无偏、低方差、低成本无法同时免费获得。 | `2605.07865`, `2606.01039`, `2606.04036`, `2606.17199` |
| 论文自述局限的统一建模 | 增量论文反复把局限指向规模、领域、teacher access、verifier/rubric 质量、compute overhead 和 hyperparameter sensitivity，但这些还没进入统一 OPD theory。 | `2605.08737`, `2605.12483`, `2605.17862`, `2605.27028`, `2606.02530`, `2606.09304` |

### B. 算法层面

| 开放问题 | 现有尝试 | 仍缺什么 |
|---|---|---|
| 自动选择 KL direction / estimator / trust region | Veto、EOPD、HPD、vOPD、PowerOPD、Trust Region OPD、SG-OPD | 从 token/state/task 到 divergence 和 estimator 的统一 controller。 |
| 何时截断、回退或 refine trajectory | Prune-OPD、Early Stopping、ADWIN、MOTAB、TRD、Near-Future Guidance | 可泛化的 supervision horizon 选择规则和 long-horizon benchmark。 |
| token / step selective OPD | TIP、TRACE、PW-OPSD、TA-OPD、Path-Conditioned SD | 结合语义角色、proof step、tool-call field、process reward 的 credit 分配。 |
| RLVR 与 OPD 调度 | SDPO、RLSD、SCOPE、Sparse-to-Dense、SSOPD、CAST、RLCSD | 何时探索、何时压缩、何时纠错、何时跳过 teacher 的统一 router。 |
| black-box dense signal 替代 | SODA、ROPD、OmniOPD、RCSD | 接近 logits/uncertainty 的 semantic distribution signal。 |
| privileged context 控制 | RLSD、TRACE、ATESD、ARG、When Context Returns | 不同 PI 类型下的 exposure policy、residual policy 和 context-return robust objective。 |
| teacher schedule / offline freshness | Lightning OPD、f-OPD、teacher isolation period | cache refresh、teacher staleness、student drift 的统一 freshness theory。 |
| 成本可控的 OPD 修复 | vOPD top-k、Early Stopping、ADWIN、OmniOPD chunk scheduling、SafeSteer localized tokens | 把 token budget、teacher forward、rollout cost、wall-clock 和最终收益放在同一评估口径下。 |

### C. 应用与评测层面

| 开放问题 | 当前缺口 |
|---|---|
| long-horizon / multi-turn / tool-use agents | 多数机制论文仍以数学 reasoning 为主；agent state、tool error、environment feedback 的 OPD fidelity 还不成熟。 |
| subjective / creative / open-ended tasks | rubric / semantic OPD 有潜力，但 rubric 校准、judge bias、style overfitting 仍明显。 |
| safety、bias、factuality | SafeSteer 和 Constitutional OPD 是新起点；self-distillation 仍可能继承 base model 风险。 |
| privacy threat model | DP-OPD 处理 private corpus，但 teacher queries、metadata、control codes、prompt identity 仍可能泄露。 |
| multimodal / cross-tokenizer / cross-family | 本文主清单不展开 MLLM/VLM 增量；跨 tokenizer、vision/action state、non-text token KL 仍是独立难题。 |
| benchmark 统一性 | OPD 方法之间 teacher/student、KL、full-vocab vs sampled-token、data regime、rollout length 不一致，横向比较困难。 |

## 可作为后续研究切入点的空白

1. **OPD 可靠性预测器**  
   把 top-k overlap、teacher entropy、supervision fidelity、initial teacher-student gap、student rollout correctness、teacher novelty 和 early KL dynamics 合成一个 go/no-go predictor。

2. **Supervision-fidelity-aware router**  
   对每个 prefix / token 决定：用 teacher KL、用 RL reward、用 student MLE、反向 teacher signal、截断 rollout、backtrack，还是跳过更新。

3. **Correction-vs-compression 在线检测**  
   判断当前 OPD update 是修正错误、压缩正确推理、强化风格，还是放大错误路径。

4. **Token teachability + semantic credit**  
   在概率 proxy 之外加入 proof step、tool-call field、error type、reasoning branch、verification token 等结构信息。

5. **Privileged context governance**  
   统一研究 teacher exposure、residual guidance、leakage bandwidth、context-return robustness 和不同 PI 类型的安全边界。

6. **Black-box semantic distribution recovery**  
   从 teacher 多样本响应、rubric decomposition、verbal uncertainty、speculative verification 中恢复近似 token/step-level uncertainty。

7. **Long-horizon OPD benchmark**  
   专门测 prefix drift、SFD、KL agreement trap、trajectory refinement、adaptive window、backtracking，而不只看最终 AIME/MATH accuracy。

8. **OPD/RLVR 方向-力度组合实验**  
   系统拆分“更新方向”和“更新力度”：RLVR 决定方向 + OPD 调力度、OPD 决定方向 + RLVR 调力度、teacher signal 反向使用、zero-advantage branch 等，判断 OPD 收益到底来自哪里。

## 一句话图谱

OPD 研究线索可以概括为：

```text
off-policy KD exposure bias
  -> student-state on-policy supervision
  -> KL direction / strict imitation / teacher-student gap failures
  -> reward scaling, adaptive divergence, trust region, estimator stabilization
  -> privileged/self-teacher information asymmetry
  -> direction-magnitude decoupling, exposure control, residual guidance
  -> long-trajectory prefix drift and supervision fidelity decay
  -> token teachability, path-conditioned credit, selective distillation
  -> semantic/logit-free/rubric/safety OPD
  -> unified OPD/RLVR routing and reliability prediction
```

目前机制最清楚的链条包括：

- **PI leakage -> RLSD / TRACE / ATESD / ARG**
- **teacher entropy / KL direction -> EOPD / HPD**
- **offline teacher inconsistency -> Lightning OPD**
- **OPSD compression-not-correction -> post-RL compaction / Sparse-to-Dense**
- **prefix drift / SFD / KL agreement trap -> Prune-OPD / SFD-aware filtering / trajectory refinement**
- **sampled-token estimator variance -> vOPD / PowerOPD**
- **token redundancy and unteachability -> TRACE / TIP / TA-OPD / PW-OPSD**

最缺的仍是统一的 **OPD 适用条件预测器**、**teacher signal reliability estimator**、**token-level causal credit assignment**、**black-box dense teacher signal** 和 **long-horizon agentic OPD 理论**。
