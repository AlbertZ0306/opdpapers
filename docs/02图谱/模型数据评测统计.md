来源：基于 `docs/02图谱/失败模式映射.md`、`docs/02图谱/基线与Loss对照.md` 中的早期论文与 2026-05+ 增量 59 篇清单，并使用本地 `papers/llm_opd_methods/` 下 PDF 抽取实验设置整理。未进行 web search。

## 统计口径

- 只统计论文实验中实际使用的模型、训练数据、验证数据和评测基准；不统计 Related Work 中引用的模型或数据集。
- `模型` 一列优先写 teacher / student；self-distillation 中 teacher 与 student 共享参数时会注明“同模型自教师”或 privileged teacher context。
- `训练数据` 指用于 SFT、RL、OPD、OPSD、SDPO 等训练阶段的数据；有些论文把 benchmark 的 train split 也作为训练数据。
- `评测基准（Benchmark）` 指最终报告性能或诊断曲线的评测集；少数论文同时用某个数据集的 train/test split，表中会分开说明。
- 综述、taxonomy 或纯诊断论文如果没有统一训练实验，标注为 N/A 或“诊断/分析”。
- 2026-05+ 增量 59 篇使用本地 PDF 快速抽取并人工去除 Related Work 噪声；若正文没有稳定抽到训练数据名，则写“未明确”。

## 总体观察

- **Qwen 系列几乎是默认底座**：除 `2604.04461` DP-OPD 主要使用 GPT-2 / DistilGPT-2 外，其余有实证实验的论文大多使用 Qwen2、Qwen2.5、Qwen3、Qwen3-VL 或 Qwen-Coder 系列。
- **数学推理是最密集的评测场景**：AIME 2024/2025、AMC 2023、HMMT、MATH500、Minerva、OlympiadBench 在 OPD 论文中反复出现。
- **2026-05+ 增量论文进一步强化了 Qwen3 + 数学 benchmark 的集中度**：增量 59 篇中，大多数实证论文使用 Qwen3 系列，并在 AIME、AMC、HMMT、MATH500 上报告结果；DeepSeek-R1-Distill-Qwen、OLMo-3、Llama、Gemma、Nemotron 多用于泛化、对照或特殊任务。
- **代码与 agentic/tool-use 是第二梯队**：HumanEval/MBPP/LiveCodeBench 用于代码生成；ToolAlpaca、BFCL、tau-Bench、Travel Planner、VitaBench、DeepPlanning 等用于 tool-use 或 agentic planning。
- **多模态 OPD 主要围绕 Qwen-VL**：相关论文使用 Qwen3-VL、Qwen2.5-VL、Pixel-Reasoner 等，评测 MathVista、MathVision、WeMath、MMMU、DocVQA、InfoVQA 等。
- **训练数据集中在几个来源**：DAPO-Math-17K、DeepMath、OpenThoughts/OpenThoughts3、OpenR1-Math、WizardCoder、Codeforces、LiveCodeBench、ToolAlpaca、LMSYS-Chat 等。

## 一句话总结

这批 OPD 方法论文的实验生态高度集中在 **Qwen 系列 + 数学推理 benchmark**，2026-05+ 增量论文进一步强化了这一点：AIME/MATH/HMMT/AMC 仍是主评测闭环，LiveCodeBench/HumanEval/MBPP 是代码扩展的主线，ToolAlpaca/BFCL/VitaBench 等承担 agentic/tool-use 扩展。训练数据上，DAPO-Math、DeepMath/DeepScaleR、OpenThoughts/OpenThoughts3、OpenR1-Math 和各类代码/tool-use 数据集承担了大多数 OPD / RLVR / self-distillation 实验；新增的 safety、rubric、context-internalization 论文开始引入 HealthBench、MedMCQA、HarmBench、SafetyBench 等更语义化的评测。
