## 依据标记

- **显式 Limitation/Future**：论文有明确 `Limitations`、`Future Work` 或同名章节。
- **Conclusion/Discussion**：论文未单列 limitation，但在结论或讨论中给出适用边界、开放方向或方法条件。
- **实验边界归纳**：论文没有清晰自述局限，以下内容来自实验范围、方法假设、消融缺口和 failure analysis 的归纳。
- **综述/分析型**：不是新算法论文，按综述或分析框架的覆盖边界总结。

## 跨论文汇总

| 主线 | 增量论文反复暴露的局限 | 对后续 OPD 算法的启发 |
|---|---|---|
| 长轨迹监督 | 很多方法都证明 full rollout 后缀 teacher signal 会变差，但 detector 仍依赖 overlap、KL、OT、freshness、entropy 等 proxy；这些 proxy 还不能统一刻画“是否可纠错”。 | 需要统一的 **prefix recoverability / supervision fidelity detector**，并能在 OPD、RLVR、backtracking、trajectory refinement 之间路由。 |
| sampled-token / estimator | vOPD、PowerOPD、OPD+、SDPG、SG-OPD 都在修正 sampled-token OPD 的方差、无界 log-ratio、梯度实现或 sign 可靠性，但尚未形成 full-vocab、top-k、sampled-token estimator 的统一理论。 | 需要区分 **显式 KL loss**、**score-function reward**、**reweighting signal** 三类实现，并给出何时无偏、何时只是启发式近似。 |
| teacher signal reliability | 单 teacher、多 teacher、privileged teacher、feedback teacher、rubric teacher 都会出现不可靠信号，局限集中在 access、calibration、style drift、PI leakage、context-return。 | OPD 的下一步不是简单增强 teacher，而是设计 **teacher signal reliability model**，判断信号能否用于方向、力度、过滤或只作 regularizer。 |
| credit granularity | 新增论文从 token 扩展到 span、step、path、trajectory、position、skill、context，但粒度选择常依赖任务和手工 proxy。 | 需要自适应 credit granularity：token-level 处理局部错误，span/step 处理 reasoning move，trajectory-level 处理 prefix failure。 |
| OPD 与 RLVR 关系 | 多数方法承认 dense teacher signal 与 sparse verifier reward 各有边界；仅靠 OPD 不一定纠错，仅靠 RLVR credit assignment 稀疏。 | 值得系统研究 **方向-力度解耦**：RLVR/OPD 分别提供更新方向、token-level 幅度、过滤信号或 regularizer 的各种组合。 |
| 黑盒 / 语义信号 | Rubric、language feedback、constitution、speculative verification 让 OPD 不再依赖 logits，但信号质量和目标等价性更难证明。 | 需要建立 logit-free OPD 的评测和理论：语义 teacher signal 何时等价于 dense KL，何时只是 reward shaping。 |
