## OPD 组件口径

- **轨迹/状态采样**：student rollout 如何产生、是否 on-policy、是否只取正确/错误轨迹、是否做数据平衡。
- **教师信号来源**：外部 teacher、自身 privileged teacher、context-conditioned teacher、rubric、multi-agent debate、feedback-conditioned teacher 等。
- **目标分布/散度**：reverse KL、forward KL、JSD、混合 KL、中间 target、likelihood-ratio objective 等。
- **权重/credit assignment**：token 选择、per-token 权重、advantage 区域、正确/错误路径路由、reward scaling。
- **训练 pipeline**：SFT/RLVR/OPD 的先后关系、offline OPD、privacy loop、post-RL compaction。

| `2602.12275` | On-Policy Context Distillation for Language Models | Context distillation 要把 prompt / historical traces / system prompt 中的知识内化到参数；off-policy 或固定数据训练不能保证 student 在自己轨迹上学会使用这些知识。 | **中**。归因到 context-conditioned teacher 与普通 student 在 student-visited states 上的分布差，需要在 student 自己的轨迹上校正。 | **教师信号来源 + 轨迹采样**。提出 OPCD：student 生成自己的 trajectories，teacher 额外看到 context / prompt / historical solution traces，对 student rollout 做 reverse KL。 |
| `2602.22495` | Reinforcement-aware Knowledge Distillation for LLM Reasoning | 把 KD 和 RL 直接结合时容易出现 distribution mismatch 与 objective interference：teacher KL regularizer 会和 reward maximization 抢目标，需要手动 loss balancing。 | **中-高**。归因到 teacher supervision 不一定对当前 policy update 有益，且普通 KL 不懂 advantage / trust region。 | **训练目标 + 权重机制**。提出 RLAD/TRRD，用 PPO/GRPO 风格 likelihood ratio objective 替代 teacher-student KL；anchor 到 teacher-old-policy mixture，实现 advantage-aware、trust-region-bounded 的 selective imitation。 |
| `2603.07079` | Entropy-Aware OPD of Language Models | 标准 reverse KL 的 mode-seeking 性质在 teacher entropy 高时会压缩多样性、产生不稳定 learning signal。 | **高**。明确把失败归因到 divergence direction 与 teacher uncertainty 不匹配：高熵 token 不该只追 teacher mode。 | **散度选择 + token 条件路由**。提出 Entropy-Aware OPD：teacher entropy 高时加入 forward KL 以覆盖合理输出，低熵时保留 reverse KL 的精确 imitation。 |
| `2603.11137` | Scaling Reasoning Efficiently via Relaxed OPD | OPD 在 reasoning transfer 中容易不稳定和 negative transfer，原因是标准 OPD 对 teacher 的 strict imitation 太硬。 | **中-高**。将 OPD 解释为 policy optimization，其中 teacher-student log-likelihood ratio 可视作 token reward。 | **reward shaping + token sampling + 训练阶段**。提出 REOPOLD：mixture-based reward clipping、entropy-based token-level dynamic sampling，以及 exploration-to-refinement 的统一训练策略，放松 teacher imitation 约束。 |
| `2604.00626` | A Survey of OPD for Large Language Models | 综述指出 off-policy/static KD 的结构性失败是 exposure bias：student 训练在 flawless teacher prefixes 上，推理却沿自己的错误轨迹前进，长序列任务中错误会累积。 | **高，整理型**。把 OPD 统一为 student-sampled trajectories 上的 `f`-divergence minimization，并整理 divergence、signal source、stabilization 三个轴。 | **无新算法**。贡献是 taxonomy：把 OPD 的核心改动定位为从 teacher-generated text imitation 转向 teacher feedback on student-produced trajectories。 |
| `2604.03128` | Self-Distilled RLVR | OPSD 的分布匹配目标在信息不对称下是 ill-posed：teacher 看到 privileged information / reference trace，student 推理时看不到；若直接匹配 `P_T(. given x,r,y_<t)`，会出现 privileged-information leakage、KL stagnation 和后期性能下降。 | **很高**。论文将 OPSD loss 分解为理想边际目标加不可约互信息 gap：`L_OPSD = L* + I(Y_t; R given X,Y_<t)`；per-sample gradient 中的 `r`-specific deviation 会把 privileged teacher evaluation 注入更新方向，形成 leakage bandwidth 和 online teacher drift。 | **更新方向/幅度解耦**。提出 RLSD：RLVR / verifier outcome 决定 token 更新方向，privileged teacher 的 evidence ratio 只作为 stop-gradient magnitude multiplier 调节 token-level credit，避免 PI 决定梯度方向。 |
| `2604.03873` | SODA: Semi On-Policy Black-Box Distillation | 黑盒 KD 面临两难：off-policy sequence KD 难纠正 student 自身错误；fully on-policy/adversarial distillation 能纠错但训练不稳且成本高。 | **中**。归因到小 student 的 zero-shot 输出天然弱于 frontier teacher，可作为稳定的 contrastive negative，不必反复动态 rollout。 | **轨迹/数据 regime**。提出 SODA：用 teacher optimal response 与一次性 static student outputs 构造 contrastive alignment signal，实现 semi-on-policy 黑盒蒸馏，减少 adversarial instability 和开销。 |
| `2604.04461` | DP-OPD | 私有语料适配中，DP-SGD 会带来 utility loss，长 autoregressive rollout 中噪声会放大 exposure bias；已有私有蒸馏要么训练 DP teacher 成本高，要么依赖 DP synthetic text pipeline。 | **中**。归因到 privacy noise 与 long rollout compounding error，以及 DP teacher / synthetic data pipeline 的训练结构负担。 | **训练 pipeline + privacy placement**。提出 DP-OPD：只对 student 做 DP-SGD，frozen teacher 在 student-generated trajectories 上给 dense token targets，避免 DP teacher 和 offline synthetic generation。 |
| `2604.10688` | SCOPE | RL 的 outcome reward 稀疏，OPD 的 dense KL 能缓解 credit assignment，但普通 OPD 对所有 rollouts 均匀施加 teacher supervision，忽略正确/错误轨迹的信号质量差异。 | **中-高**。归因到 incorrect rollout 中 teacher 是否真正能纠错、correct rollout 中 student 是否仍低置信，这两类信号应分别加权。 | **轨迹路由 + 权重机制**。提出 dual-path adaptive weighting：错误轨迹用 teacher-perplexity-weighted KL，正确轨迹用 student-perplexity-weighted MLE，并做 group-level normalization。 |
| `2604.12002` | Self-Distillation Zero | RLVR 的 binary reward 泛用但稀疏；distillation 有 dense token signal 但通常需要外部 teacher 或高质量 demonstrations。 | **中-高**。通过 ablation 指出 reviser 可做 token-level self-localization，且定期 teacher synchronization 能形成 iterative self-evolution。 | **教师信号来源 + dense supervision 构造**。提出 SD-Zero：同一模型分 Generator/Reviser，Reviser 看到初始回答和 binary reward 后生成改进回答，再把 Reviser 的 token distribution 蒸馏回 Generator。 |
| `2604.13010` | Lightning OPD | 标准 OPD 需要 live teacher server，基础设施开销大；naive offline OPD 预计算 teacher log-probs 后复用却不能稳定匹配在线 OPD。 | **高**。把 naive offline 失败归因到 teacher consistency：SFT 与 OPD 阶段若不是同一 teacher，会引入 gradient bias。 | **训练 pipeline + teacher consistency**。提出 Lightning OPD：在满足 teacher consistency 的 SFT rollouts 上预计算 teacher log-probs，离线复用；证明与标准 OPD 共享 optimum，并约束 gradient discrepancy。 |
| `2604.13016` | Rethinking OPD | OPD 并非稳定“免费午餐”：teacher 更强但 thinking pattern 与 student 不兼容，或 teacher 没提供 student 未见过的新能力时，OPD 会失败；弱到强反向蒸馏也可能看似无效。 | **很高**。用现象学实验、weak-to-strong reverse distillation、token-level probing 归因到 thinking-pattern compatibility、teacher novelty，以及 student-visited states 上 high-probability token alignment。 | **数据/初始化 recipe**。不是单一新算法，改的是 OPD 前置条件：off-policy cold start 先对齐 thinking pattern；teacher-aligned prompt selection 选择 teacher 真有新知识且和 student 状态可对齐的数据。 |
| `2604.14084` | TIP: Token Importance in OPD | 全 token OPD 成本高，而且“哪些 token 真有用”理解不完整；只按 entropy 选 token 会漏掉过度自信但错误的 token。 | **高**。把 token 价值归因到两类区域：高 student entropy 的不确定 token，以及低 entropy + 高 teacher-student divergence 的 overconfident-wrong token。 | **token selection + memory efficiency**。提出 TIP 两轴 taxonomy，用 student entropy 和 teacher-student divergence 做 type-aware token selection；只训练少量关键 token 也接近或超过 full-token OPD。 |
| `2604.20244` | Hybrid Policy Distillation for LLMs | KD/OPD 效果受 divergence direction、optimization strategy、data regime 交织影响；单用 forward 或 reverse KL 会在 mode coverage 与 mode seeking 间取舍，稳定性和效率受限。 | **中**。把不同 KD 方法统一为 token-level reweighted log-likelihood objective，说明散度方向和数据 regime 决定优化行为。 | **散度混合 + 数据 regime**。提出 HPD：混合 forward/reverse KL，同时结合 off-policy data 与 lightweight approximate on-policy sampling，平衡覆盖、多样性和精确 imitation。 |
| `2605.01347` | MAD-OPD | 单 teacher OPD 有 capability ceiling：teacher 错了 student 就继承错；agentic tasks 中 step-level errors 会在长轨迹里累积，训练不稳。 | **中-高**。归因到 teacher signal 的单点错误和 agentic trajectory 的多步误差累积；不同任务适合不同 divergence。 | **教师集合 + step-level sampling + divergence choice**。提出 MAD-OPD：多个 teacher 围绕 student on-policy state debate，以 post-debate confidence 加权 token supervision；OPAD 加 step-level sampling；agentic 用 JSD，code 用 reverse KL。 |
| `2605.03677` | Uni-OPD | OPD 虽然有效，但可靠提升条件不清楚；两大瓶颈是 student 没充分探索 informative states，以及 teacher 对 student rollouts 的监督不可靠。 | **高**。从 student perspective 和 teacher perspective 双侧归因：数据状态不够 informative，token-level guidance 与 outcome reward order 不一致。 | **状态探索 + teacher margin calibration**。提出 Uni-OPD：用数据平衡策略促进 student exploration；用 outcome-guided margin calibration 恢复正确/错误轨迹间的 order consistency。 |
| `2605.06188` | OPSD Compresses What RLVR Teaches | OPSD 在 thinking-enabled 数学推理中承诺的精度提升变小甚至为负；它不一定能纠正失败轨迹。 | **高**。把 OPSD 拆到 correct / incorrect rollout groups 分别观察，发现 OPSD 更像压缩正确长推理，而不是修正错误长推理；错误轨迹上训练会伤害 accuracy。 | **训练 pipeline + 轨迹筛选**。建议 SFT -> RLVR -> OPSD，把 OPSD 作为 post-RL compaction stage；主要在 correct rollouts 上压缩冗余思考，而不把它当 correction mechanism。 |
| `2605.06387` | Asymmetric On-Policy Distillation | 标准 OPD 的 advantage-weighted policy gradient 有三类结构性弱点：高方差更新、zero-advantage 区域梯度消失、纠错信号不足时探索受限。 | **中-高**。归因到 positive / non-positive advantage token 不应使用对称处理，负强化在弱信号区域效率低。 | **advantage 区域目标函数**。提出 AOPD：正 advantage 仍做 RL exploit；非正 advantage 区域用 localized teacher divergence matching 替代无效负强化，同时保持更高 policy entropy。 |
| `2605.06597` | UniSD | LLM self-distillation 中，自生成轨迹自由度高、正确性任务相关，貌似合理的 rationale 也可能给出不稳定/不可靠监督；已有方法只研究孤立组件。 | **中-高**。系统分析 supervision reliability、representation alignment、training stability 三类因素，以及多组件交互。 | **self-distillation 组件框架**。提出 UniSD：集成 multi-teacher agreement、EMA teacher stabilization、token-level contrastive learning、feature matching、divergence clipping，形成 UniSDfull pipeline。 |
| `2605.07396` | Rubric-based On-policy Distillation | OPD 依赖 teacher logits，限制在 white-box teacher；很多 proprietary teacher 只能给 responses，不能给 logits。 | **中**。归因到 teacher signal 表示形式太底层：logit-level supervision 可被 prompt-specific semantic rubric 替代。 | **教师信号表示**。提出 ROPD：从 teacher-student contrasts 诱导 prompt-specific rubrics，再用 rubric 给 student rollouts 打分做 on-policy optimization，实现 black-box-compatible OPD。 |
| `2605.10781` | Rebellious Student | self-distillation 在 student 失败时有用，但在成功 rollouts 上会覆盖 student 自己的选择，压制它已经成功的 reasoning path。 | **高**。归因到 information asymmetry 的另一面：teacher 没预测到但 student 成功的 token 正是有价值的 self-driven reasoning。 | **teacher signal 符号/使用方式**。提出 RLRT：在正确轨迹上反向读取 teacher signal，强化 student 成功但偏离 privileged teacher 的 token，把 teacher mismatch 变成探索信号。 |

## 一句话结论

早期 26 篇中，真正走完“发现 failure mode -> 机制归因 -> 对症改算法”的论文主要集中在 `2604.13016`、`2604.03128`、`2603.07079`、`2604.13010`、`2604.14084`、`2605.06188`、`2605.06387`、`2605.10781`。2026-05+ 增量后，OPD failure mode 明显进一步集中到五条主线：**长轨迹 prefix drift / supervision fidelity decay**，**sampled-token KL 方差和 trust-region 稳定性**，**token teachability / step credit / selective distillation**，**privileged context 泄漏与 context-return 鲁棒性**，以及 **logit-free / rubric / feedback 等语义 teacher signal**。也就是说，新增论文不再只是“换一个 teacher 或 KL”，而是在系统拆解 OPD 的 state、trajectory、token、teacher-signal reliability 和 update geometry。
