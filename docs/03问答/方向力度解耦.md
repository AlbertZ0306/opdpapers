## Q: OPD 与 RLVR 的方向-力度解耦，已有论文是否做过类似实验分析？

用户提出的核心矩阵是：

| 更新方向 | 更新力度 | 例子 |
| --- | --- | --- |
| RLVR direction | RLVR magnitude | GRPO baseline |
| OPD direction | OPD magnitude | standard sampled-token OPD |
| RLVR direction | OPD magnitude | teacher 只调 token credit |
| OPD direction | RLVR magnitude | verifier 决定是否信 teacher |
| hybrid direction | hybrid magnitude | router / gating |

这里的“方向”和“力度”可以统一写成：

$$
\Delta \theta
\approx
\mathbb{E}_{i,t}
\left[
c_{i,t}\,
g_{i,t}
\right].
$$

其中：

- `g_{i,t}` 是方向基向量。例如 sampled-token policy gradient 里的 `\nabla_\theta \log \pi_\theta(a_{i,t}\mid s_{i,t})`，或者 full-vocabulary KL 里的分布匹配梯度。
- `c_{i,t}` 是力度、权重或 advantage。例如 GRPO 的 outcome advantage、OPD 的 teacher-student log-ratio、KL 权重、token mask、sign gate、clipping 系数。

## A: 总体结论

**已有论文做了很多局部类似实验，但还没有看到一篇论文完整做出上述 2 轴全组合矩阵。**

最接近的几条线是：

1. **RLSD / RLCSD / SC-GRPO**：基本是在做 `RLVR direction + OPD magnitude`。也就是 verifier / outcome reward 决定正负方向，teacher / KL 只决定 token-level credit 的强弱。
2. **CAST / SG-OPD / AOPD**：开始允许 teacher signal 在局部区域改变方向，或者根据 RLVR 与 OPD 的 sign consistency 做 gate。这已经接近 `hybrid direction + hybrid magnitude`。
3. **SDPG / OPD+ / PowerOPD**：主要澄清 OPD signal 作为 explicit KL loss、policy-gradient reward、bounded reward transform 时，真实梯度和方差不一样。这支持“方向-力度”视角，但不是完整 RLVR x OPD 组合实验。
4. **RLRT**：在成功 rollout 上反向使用 teacher signal，说明 teacher-student gap 不只能用于“模仿 teacher”，也能作为 self-driven exploration 的力度信号。

所以，这个方向不是空白，但仍有明确缺口：**还缺一个在同一 rollout、同一 verifier、同一 teacher、同一 compute budget 下，把 direction source 和 magnitude source 系统拆开的 factorial ablation。**

## 这些论文真正证明了什么？

### 1. `RLVR direction + OPD magnitude` 已经被较充分验证

RLSD、RLCSD、SC-GRPO 三篇的共同结论是：

$$
\tilde{A}_{i,t}
=
A_i \cdot w^{\mathrm{OPD}}_{i,t},
\quad
w^{\mathrm{OPD}}_{i,t} > 0.
$$

因为 `w_{i,t}` 始终为正，所以 token update 的正负号仍由 `A_i` 决定：

$$
\mathrm{sign}(\tilde{A}_{i,t})
=
\mathrm{sign}(A_i).
$$

这意味着 teacher / KL 不决定“奖还是罚”，只决定“奖罚多大”。RLCSD 的 `-AORM anchoring` 消融和 SC-GRPO 的 additive KL loss 对照，都支持这个设计。

### 2. 只做 magnitude-only 可能过于保守

CAST 指出，如果整条轨迹是正确的，并不意味着每个 token 都应正向强化；如果整条轨迹是错误的，也不意味着每个 token 都应负向压制。因此它允许局部 sign reversal：

$$
A^{\mathrm{CAST}}_{i,t}
=
\mathrm{router}
\left(
R_i,\,
A_i,\,
g^{T-S}_{i,t}
\right),
$$

其中 `R_i` 是 verifier correctness，`A_i` 是 GRPO base advantage，`g^{T-S}_{i,t}` 是 teacher-student gap。CAST 的 Table 17 事实上已经在做从 RLSD 到 hybrid sign routing 的递进实验。

### 3. teacher signal 不一定是 imitation signal

RLRT 的核心是：在成功 rollout 上，如果 student 选择了 teacher 不看好的 token，但最后成功，这个 token 可能代表 self-driven successful reasoning。于是 teacher gap 可以被反向使用：

$$
w^{\mathrm{RLRT}}_{t}
\propto
\left(
\frac{\pi_S(a_t\mid s_t)}
{\pi_T(a_t\mid s_t)}
\right)^{\mathrm{sign}(A)}.
$$

这说明 OPD signal 的好处不一定来自“靠近 teacher direction”，也可能来自“识别 student 自己已经找到的有效路径，并调高其 magnitude”。

### 4. explicit KL loss、token reward、token reweighting 的梯度含义不同

SDPG、OPD+、PowerOPD 都在强调：把 KL 作为 loss、作为 score-function reward、作为 token weight，不一定等价。

例如 sampled-token 形式常见为：

$$
\mathbb{E}
\left[
A^{\mathrm{KL}}_{t}
\nabla_\theta \log \pi_\theta(a_t\mid s_t)
\right].
$$

而 full-vocabulary KL 是：

$$
\nabla_\theta
D_{\mathrm{KL}}
\left(
\pi_\theta(\cdot\mid s_t)
\|
\pi_T(\cdot\mid s_t)
\right).
$$

二者的方向基向量不同：前者只沿 sampled token 的 likelihood direction 更新，后者沿整个词表分布差更新。方向-力度矩阵实验必须把这两个轴分开，否则会把 estimator difference 误认为 direction/magnitude difference。

## 仍然缺的关键实验

### 实验 1：真正的 2 x 2 direction/magnitude factorial

在同一批 on-policy rollouts 上，固定 teacher、verifier、optimizer、token budget，只替换 advantage 构造：

| 编号 | direction source | magnitude source | 形式示意 |
| --- | --- | --- | --- |
| A | RLVR | RLVR | `A_i * grad log pi(a_t)` |
| B | OPD | OPD | `grad D_KL(pi_theta, pi_T)` 或 sampled-token OPD reward |
| C | RLVR | OPD | `A_i * f(Delta^T-S_{i,t}) * grad log pi(a_t)` |
| D | OPD | RLVR | `m(A_i) * g_OPD(s_t)`，或 `1[A_i > 0] * grad D_KL(pi_theta, pi_T)` |
| E | hybrid | hybrid | `router(A_i, R_i, Delta^T-S_{i,t}, conflict_t)` |

其中 D 是当前文献最薄弱的一格。它要测试：**如果 teacher 决定分布方向，而 verifier 只决定信任强度或 mask，是否比 RLVR direction + OPD magnitude 更好？**

### 实验 2：sign 和 scale 分离

对每个 token 的 effective advantage 做拆解：

$$
\tilde{A}_{i,t}
=
s_{i,t}
\cdot
m_{i,t},
\quad
s_{i,t}\in\{-1,0,+1\},
\quad
m_{i,t}\ge 0.
$$

分别测试：

- `s_{i,t}` 来自 RLVR，`m_{i,t}` 来自 RLVR；
- `s_{i,t}` 来自 RLVR，`m_{i,t}` 来自 OPD；
- `s_{i,t}` 来自 OPD sign，`m_{i,t}` 来自 RLVR；
- `s_{i,t}` 来自 sign-consistency gate，`m_{i,t}` 来自 KL gap / entropy / confidence；
- conflict token 直接 mask、interpolate、reverse、或交给 verifier/teacher router。

这会比只比较最终 loss 更清楚地回答“OPD 的收益来自方向还是力度”。

### 实验 3：按 rollout 区域分层

论文已经暗示，不同区域最优组合不同：

| 区域 | 可能更适合的组合 | 相关论文依据 |
| --- | --- | --- |
| mixed group，reward 有方差 | `RLVR direction + OPD magnitude` | RLSD、RLCSD、SC-GRPO |
| all-correct / all-wrong，GRPO advantage 归零 | hybrid / CAST-style bounded base advantage | CAST |
| correct rollout 内 teacher-negative token | 局部 OPD negative direction 或 RLRT self-driven positive direction，需要区分 teacher 是否可靠 | CAST、RLRT |
| incorrect rollout 内 teacher-positive token | bounded positive local credit，但必须 verifier-grounded | CAST |
| teacher/student sign conflict | gate、interpolate、filter | SG-OPD |
| non-positive advantage / zero-gradient 区域 | teacher distribution matching 可补方向 | AOPD |

### 实验 4：控制 OPD 粒度

同一个 direction/magnitude 组合下，还要控制 OPD signal 的粒度：

- full-vocabulary KL；
- top-k / subset KL；
- sampled-token KL estimator；
- explicit KL loss；
- token-level reward；
- token-level reweighting。

否则可能出现混淆：一个方法看起来是“direction 更好”，其实只是 full-vocab KL 方差更低；另一个方法看起来是“magnitude 更好”，其实只是 sampled-token estimator 更 noisy。

## 可以形成的新研究问题

1. **OPD 的收益主要来自哪里？**
   - teacher 给了更好的分布方向；
   - teacher 给了更细的 token-level magnitude；
   - teacher 只是 regularizer / trust region；
   - 或者三者在不同 rollout 区域分别成立。

2. **何时应该让 teacher 决定方向？**
   - 当 verifier advantage 为 0；
   - 当 teacher-student sign 与 verifier sign 一致；
   - 当 token teachability / reliability 高；
   - 当 teacher signal 不是 privileged leakage 或 style token。

3. **何时只应该让 teacher 调力度？**
   - privileged information 可能泄漏；
   - teacher/student gap 主要反映 style；
   - outcome reward 可靠但稀疏；
   - teacher signal 有帮助但 wrong-sign 风险高。

4. **能否统一 RLSD、RLRT、CAST、RLCSD、SG-OPD？**
   可以把它们看成同一个 router family：

$$
\tilde{A}_{i,t}
=
\mathrm{Router}
\left(
A^{\mathrm{RLVR}}_i,\,
R_i,\,
\Delta^{T-S}_{i,t},\,
\mathrm{Reliability}_{i,t},\,
\mathrm{Conflict}_{i,t}
\right).
$$

其中：

- RLSD：router 保持 RLVR sign，只调 magnitude；
- RLCSD：先 contrastive cleaning，再保持 RLVR sign；
- SC-GRPO：KL 作为 GRPO gradient 的 multiplicative weight；
- RLRT：在 correct rollout 上反向解释 teacher gap；
- CAST：按 correctness 和 gap sign 做局部 sign reversal；
- SG-OPD：按 RLVR/OPD sign consistency gate；
- AOPD：在 non-positive / zero-gradient 区域切到 teacher distribution matching。

## 最终判断

这个方向已经有较强相关文献基础，尤其是 RLSD、RLCSD、SC-GRPO、CAST、SG-OPD、SDPG。但现有工作大多是各自提出一个修复算法，然后做自身组件消融；**还没有形成一个统一的 direction-source x magnitude-source 对照实验框架**。

因此，“OPD 的好处究竟来自更新方向、token-level 力度，还是 regularization”仍然是一个可以继续系统研究的问题。最有价值的切入点不是再提出一个单点加权公式，而是做一个受控实验框架，把 RLVR / OPD / hybrid 的 sign、scale、粒度、估计器和 router 分开测。
