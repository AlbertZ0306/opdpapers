## 2601.07155 - Stable On-Policy Distillation through Adaptive Target Reformulation

本地文件：`../papers/llm_opd_methods/2026_kaist_stable_on_policy_distillation_through_adaptive_target_reform_2601.07155.pdf`

### Q: forward KL 和 reverse KL 的区别是什么？这篇论文的 loss 是怎么设计的？

**Forward KL** 以 teacher 分布为期望：

$$
D_{\mathrm{KL}}(P_T \| P_S)
= \sum_y P_T(y)\log\frac{P_T(y)}{P_S(y)}
$$

它是 teacher 视角：teacher 认为可能的 token，student 也必须给概率。优点是覆盖 teacher 的多个模式；缺点是如果 `P_T(y)>0` 但 `P_S(y)` 很小，惩罚会很大，早期弱 student 上可能产生 pathological gradients。

**Reverse KL** 以 student 分布为期望：

$$
D_{\mathrm{KL}}(P_S \| P_T)
= \sum_y P_S(y)\log\frac{P_S(y)}{P_T(y)}
$$

它是 student 视角：student 只关心自己给概率的 token 是否被 teacher 支持。优点是数值更稳；缺点是 mode-seeking，student 可以只押 teacher 的某个高概率模式，忽略其它合理模式，导致 diversity collapse。

这篇论文的核心不是直接用 teacher `P_T` 当目标，而是构造一个 teacher 和 student 之间的中间目标分布 `Q_beta`：

$$
z_T = \log P_T(\cdot \mid x, y_{<t}), \qquad
z_S = \log P_S(\cdot \mid x, y_{<t})
$$

$$
Q_\beta(\cdot \mid x, y_{<t})
\propto \exp(z_T + \beta z_S)
= \frac{P_T(\cdot \mid x, y_{<t})\,P_S(\cdot \mid x, y_{<t})^\beta}{Z}
$$

然后在 student 自己生成的 on-policy trajectory 上逐 token 训练：

$$
y \sim P_S(\cdot \mid x), \qquad
L = \sum_t L_t
$$

其中可以用 forward KL：

$$
L_t = D_{\mathrm{KL}}(Q_\beta \| P_S)
$$

也可以用 reverse KL：

$$
L_t = D_{\mathrm{KL}}(P_S \| Q_\beta)
$$

论文主线方法是 **Veto / Adaptive Target Reformulation**：`beta=0` 时 `Q_beta=P_T`，退化为普通 OPD；`beta>0` 时，目标分布会被 student 当前分布调制。如果某个 token teacher 很喜欢但 student 当前几乎不给概率，`P_S(.)^beta` 会把它在 `Q_beta` 中的权重压低，避免 forward KL 强行让早期 student 追一个它完全不会的 token。

因此 `beta` 有两个作用：

- 在 forward KL 下，它像 **gradient veto**：student 当前低置信 token 会削弱 teacher 过硬监督，缓解 pathological gradients。
- 在 reverse KL 下，展开后会改变 student entropy 项，像 **decisiveness knob**：`beta` 越大，student 越 decisive / mode-seeking；`beta` 越小，多样性保留越强。

对 OPD 组件的定位：这篇论文主要改的是 **目标分布/散度层**，不是换 teacher、不是换 rollout，而是在 teacher target 与 student current policy 之间构造一个 adaptive intermediate target，从而稳定大 teacher-student gap 下的 OPD。

## 2601.20802 vs 2601.18734 - SDPO 和 OPSD 的区别

本地文件：

- `../papers/llm_opd_methods/2026_ethzurichlas_reinforcement_learning_via_self_distillation_2601.20802.pdf`
- `../papers/llm_opd_methods/2026_ucla_self_distilled_reasoner_on_policy_self_distillation_for_larg_2601.18734.pdf`

### Q: “RLVR 只从每次尝试拿到 scalar outcome reward，严重限制 token-level credit assignment；很多环境其实有 runtime error、judge feedback 等富文本反馈没有被用起来”是什么意思？

在 `2601.20802 Reinforcement Learning via Self-Distillation` 里，作者批评的是标准 RLVR 的信息瓶颈。

标准 RLVR 可以抽象成：

$$
y \sim \pi_\theta(\cdot \mid x), \qquad r = R(x, y)
$$

其中 `x` 是题目，`y` 是模型生成的一次完整回答，`r` 是环境返回的结果分数。很多任务里 `r` 只是一个 scalar outcome reward，比如：

- 数学题：答案对 `1`，答案错 `0`
- 代码题：unit tests 通过 `1`，失败 `0`
- verifier：最终结论 correct / incorrect

问题是，这个 scalar 只告诉模型 **这整段回答最后对不对**，但不告诉模型：

- 哪一步开始错了
- 哪个 token / 哪行代码导致失败
- 正确方向应该怎么改
- 是逻辑错、格式错、单位错，还是 runtime error

这就是 token-level credit assignment bottleneck。模型生成了几千个 token，最后只拿到一个 `0`，很难知道应该惩罚或修正哪几个 token。

在 GRPO / RLVR 里还有一个更具体的问题：如果同一组 sampled responses 全部都错，或者全部都对，那么 group-relative advantage 可能变成 0，导致这批样本几乎没有有效 policy update。也就是说，模型花了很多采样成本，但没有得到可用学习信号。

`2601.20802` 指出，很多 verifiable environments 其实不只返回 scalar reward，还会返回更丰富的 tokenized feedback。例如代码环境可能返回：

- runtime error 类型
- 报错行号
- failing unit test
- last executed input
- memory error / index error

LLM judge 也可能返回自然语言评价，例如“单位转换错了”“没有考虑边界条件”“结论和推导矛盾”。这些反馈不只是告诉模型“错了”，还告诉模型 **为什么错**。

所以论文提出 RLRF：Reinforcement Learning with Rich Feedback。核心思想是把这些富文本反馈喂给当前模型，让当前模型在 feedback-conditioned context 下重新判断：

$$
\pi_\theta(\cdot \mid x, y, f)
$$

其中 `f` 是 runtime error / judge feedback / failing tests 等反馈。这个 feedback-conditioned model 充当 self-teacher，给原始 student policy 提供更细粒度的 next-token supervision。论文的方法叫 SDPO：Self-Distillation Policy Optimization。

一句话：**SDPO 不是只根据最终 0/1 奖励更新整段回答，而是把环境反馈转成“模型自己看完反馈后认为每一步应该怎么写”的 dense token-level signal。**

### Q: `2601.20802 Reinforcement Learning via Self-Distillation` 和 `2601.18734 Self-Distilled Reasoner` 有什么区别？

二者都属于 self-distillation：teacher 和 student 本质上是同一个模型，只是 teacher 看到额外信息，student 不看额外信息。但它们的额外信息来源、训练场景和目标不同。

| 对比维度 | `2601.18734` Self-Distilled Reasoner / OPSD | `2601.20802` Reinforcement Learning via Self-Distillation / SDPO |
|---|---|---|
| 核心场景 | reasoning dataset post-training | RL / online environment post-training |
| teacher 看到什么额外信息 | ground-truth reference solution / verified reasoning trace | 环境反馈，例如 runtime error、failing tests、LLM judge feedback |
| student 看到什么 | 只看到原题 | 原题和自己的普通 rollout，不直接依赖 feedback-conditioned prompt |
| teacher 的角色 | privileged teacher：知道参考解答后，对 student rollout 上每个 token 给分布监督 | feedback-conditioned self-teacher：看完错误反馈后，回头判断原 rollout 每步应该怎样修 |
| 主要解决的问题 | 不用外部大 teacher，也能利用 reasoning dataset 里的 reference solution 做 on-policy dense distillation | 标准 RLVR 的 scalar reward 太稀疏，无法做 token-level credit assignment |
| 训练信号 | per-token divergence between privileged teacher and student | feedback-informed next-token predictions / corrected log-probs |
| 和 RLVR 的关系 | 更像 RLVR / SFT 的替代或补充；论文也指出当前未显式使用 correctness verification | 直接嵌进 RLVR/RLRF pipeline，是对 RLVR credit assignment 的增强 |
| 关键局限 | 只实验到 8B；没有显式利用 correctness verification；问题太难时 privileged teacher 也无法有效监督 | 依赖模型 in-context learning 能力；依赖反馈质量；弱模型可能不如 GRPO；有额外 log-prob 计算开销 |

最核心区别：

- `2601.18734` 的 teacher privilege 来自 **标准答案 / 参考推理轨迹**。
- `2601.20802` 的 teacher privilege 来自 **模型自己尝试后的环境反馈**。

可以把二者理解成：

- OPSD：给 teacher 看“标准解”，让它指导 student。
- SDPO：给 teacher 看“你刚才哪里错了”，让它回头指导 student。

因此，`2601.18734` 更像是把 supervised reasoning dataset 里的参考解答转成 on-policy self-distillation；`2601.20802` 更像是把 RL 环境中的 rich feedback 转成 dense self-distillation signal。

## 2602.04942 - Privileged Information Distillation for Language Models

本地文件：`../papers/llm_opd_methods/2026_servicenowai_privileged_information_distillation_for_language_models_2602.04942.pdf`

### Q: “多轮 agentic 环境中，训练时 privileged information 可让模型成功，但推理时 student 无法访问；同时 frontier model 常隐藏 CoT，只暴露 action trajectory，破坏标准 CoT distillation”是什么意思？

这篇论文研究的是 **训练时有额外信息，推理时没有额外信息** 的蒸馏问题。论文把这种额外信息叫 privileged information，简称 PI。

在多轮 agentic 任务中，模型不是一次性回答问题，而是要连续执行动作。例如：

- 读用户请求
- 调用工具
- 根据工具返回继续判断
- 再调用下一个工具
- 最后完成任务

形式上可以理解为：

$$
s_t = \text{当前状态}, \qquad o_t = \text{下一步动作或工具调用}
$$

如果模型在训练时看到额外信息 `I`，它的 teacher policy 是：

$$
\pi_T(o \mid s, I)
$$

而推理时 student 不能看到 `I`，只能用：

$$
\pi_S(o \mid s)
$$

这就是论文说的核心 transfer problem：**如何把 `I` 带来的能力转移到不看 `I` 的 student policy 里。**

这里的 PI 可以是多种训练时信息。论文语境里尤其重要的是 frontier model 的成功 action trajectory。Frontier agent 可能成功完成任务，但很多商业或 frontier 模型不会暴露完整 Chain-of-Thought，只暴露最终可见动作，例如：

```text
Tool Call 1
Tool Call 2
Tool Call 3
...
```

也就是说，我们能看到它 **做了什么 action**，但看不到它 **为什么这么做的 reasoning / CoT**。

这会破坏标准 CoT distillation。标准 CoT distillation 通常需要这样的监督：

```text
问题 -> 推理过程 CoT -> 答案 / 动作
```

但在 frontier agent 场景中，我们只有：

```text
问题 -> action trajectory
```

没有中间 reasoning。于是 successful behavior 是可见的，但 reasoning process 不可见。论文原文也明确说：successful behavior is observable, but the reasoning process is not。

这就是那句话的意思：

- **训练时 PI 可让模型成功**：比如给 teacher 看 frontier agent 的成功工具调用轨迹，或其它额外 hints，teacher 更容易完成 long-horizon agentic task。
- **推理时 student 无法访问 PI**：部署时 student 只能根据用户输入和环境状态自己决策，不能偷看训练时的额外信息。
- **frontier model 隐藏 CoT**：不能直接拿到完整推理链做 SFT / CoT distillation。
- **只暴露 action trajectory**：只能看到工具调用序列和可见动作。
- **标准 CoT distillation 失效**：因为缺少最关键的 reasoning trace，只模仿 action 又可能学不到为什么这么做。

论文提出两个方法：

1. **`pi`-Distill**  
   同一个参数共享模型里同时训练：

   $$
   \pi_T(o \mid s, I)
   $$

   和：

   $$
   \pi_S(o \mid s)
   $$

   teacher 学会利用 PI 成功完成任务，student 同时学习在没有 PI 时复制这种高回报行为。

2. **OPSD variant**  
   用 student 和 PI-conditioned teacher 之间的 reverse KL 作为训练 penalty，在 on-policy RL 中把 PI teacher 的行为蒸馏给 student。

一句话总结：这篇论文不是普通“看答案学推理”的 CoT 蒸馏，而是在 **看不到 frontier model 内部推理、只能看到成功动作轨迹** 的情况下，研究如何把训练时额外信息带来的能力转移到推理时没有额外信息的 student 里。

## 2602.12125 - Learning beyond Teacher: Generalized OPD with Reward Extrapolation

本地文件：`../papers/llm_opd_methods/2026_tencenthunyuan_learning_beyond_teacher_generalized_on_policy_distillation_w_2602.12125.pdf`

### Q: “标准 OPD 隐含把 reward term 和 KL regularization 等权绑定，导致优化受 teacher 边界约束，尤其在 expert merging 或 strong-to-weak 蒸馏时不易超越 teacher”是什么意思？

这篇论文的核心是把 OPD 重新解释成一种 **dense KL-constrained RL**。

标准 OPD 表面上是在 student-generated trajectories 上匹配 teacher logits。但论文指出，它等价于一个带 KL 约束的 RL 目标，其中：

- reward term 来自 teacher 相对 reference 的隐式 reward
- KL regularization 限制 student 不要偏离 reference
- 标准 OPD 里这两项的相对权重被固定成 `1:1`

论文把这个问题推广成 G-OPD：

$$
J_{\mathrm{G\text{-}OPD}}(\theta)
=
\max_\theta
\mathbb{E}_{x\sim D,\,y\sim \pi_\theta(\cdot|x)}
\left[
\lambda \log \frac{\pi^*(y|x)}{\pi_{\mathrm{ref}}(y|x)}
-
D_{\mathrm{KL}}\left(\pi_\theta(y|x)\|\pi_{\mathrm{ref}}(y|x)\right)
\right]
$$

其中：

- `pi*` 是 teacher / expert policy
- `pi_ref` 是 reference model
- `lambda` 控制 reward term 相对 KL regularization 的强度

标准 OPD 对应的是：

$$
\lambda = 1
$$

所以论文说标准 OPD 隐含把 reward term 和 KL regularization 绑死了。

为什么这会受 teacher 边界约束？因为当：

$$
\lambda = 1
$$

student 的目标基本是 **恢复 teacher 的 post-training behavior**。论文实验也说，standard OPD can fully recover the post-training behavior：student 会接近 teacher 的 accuracy 和 response length。

这意味着标准 OPD 更像：

```text
让 student 模仿 teacher
```

而不是：

```text
让 student 超过 teacher
```

论文进一步分析 G-OPD 的最优解：

$$
\log \pi_\theta(y|x)
=
\lambda \log \pi^*(y|x)
+ (1-\lambda)\log \pi_{\mathrm{ref}}(y|x)
$$

当：

$$
0 < \lambda < 1
$$

student 处在 reference 和 teacher 之间。这叫 reward interpolation。

当：

$$
\lambda > 1
$$

student 不只是匹配 teacher，而是沿着 teacher 相对 reference 的提升方向继续外推：

$$
\log \pi_\theta
=
\log \pi^*
+ (\lambda - 1)(\log \pi^* - \log \pi_{\mathrm{ref}})
$$

这就是 reward extrapolation，也就是 ExOPD。

直观理解：

```text
reference -> teacher 是一次 RL/post-training 带来的能力提升方向
ExOPD 试图沿这个方向再多走一步
```

所以它可能 beyond teacher。

论文中特别讨论两个场景：

1. **Expert merging**

   多个 domain teachers 是同一个 base model 在不同领域 RL 得到的专家。标准 OPD 往往只能把 student 拉到各个 teacher 附近；ExOPD 通过 `lambda>1` 加强隐式 reward，可以把多个专家的能力合并回一个 unified student，甚至超过单个 domain teacher。

2. **Strong-to-weak distillation**

   用大 teacher 蒸馏小 student 时，如果 reference 选 student base：

   $$
   \log \frac{\pi^*}{\pi_{\mathrm{student\ base}}}
   $$

   这个 reward 可能有噪声，因为大 teacher 和小 student 的知识、能力、分布差距太大。

   论文提出 reward correction：如果能拿到 teacher pre-RL base，就把 reference 换成：

   $$
   \pi_{\mathrm{teacher\ base}}
   $$

   这样：

   $$
   \log \frac{\pi^*}{\pi_{\mathrm{teacher\ base}}}
   $$

   更像 teacher RL post-training 真实学到的 implicit reward，因此更准确。

但是论文也指出，这有代价：

- 需要访问 teacher 的 pre-RL variant
- 计算大 reference model 的 log-prob 会增加 computational cost

一句话总结：这篇论文认为标准 OPD 默认只是在“以固定强度模仿 teacher”，所以学生通常被 teacher 边界限制；G-OPD/ExOPD 通过调大 reward weight `lambda>1`，把 teacher 相对 reference 的能力提升方向继续外推，从而尝试让 student 超越 teacher。

## 2602.12275 - On-Policy Context Distillation for Language Models

本地文件：`../papers/llm_opd_methods/2026_microsoftresearch_on_policy_context_distillation_for_language_models_2602.12275.pdf`

### Q: “Context distillation 要把 prompt / historical traces / system prompt 中的知识内化到参数；off-policy 或固定数据训练不能保证 student 在自己轨迹上学会使用这些知识”是什么意思？

这篇论文讨论的是 **context distillation**：把原本只能放在上下文里的信息，压缩进模型参数里。这里的 context 可以是：

- prompt / instruction
- few-shot demonstrations
- retrieved documents
- historical solution traces 中提炼出的 experience
- optimized system prompt

普通 in-context learning 的问题是：这些知识是 transient 的。只要 context reset，模型就又失去了这些信息。Context distillation 的目标是训练一个不再需要额外 context 的 student，让它在推理时只看到输入 `x`，也能表现得像训练时看到 context `c` 的 teacher。

可以写成：

$$
\pi_{\mathrm{teacher}}(\cdot \mid c, x)
\rightarrow
\pi_\theta(\cdot \mid x)
$$

其中：

- teacher 看到 `c+x`
- student 只看到 `x`
- 蒸馏目标是让 student 的参数内化 `c` 中的知识

论文指出，已有 context distillation 通常是 **off-policy forward KL on a fixed dataset**。也就是先用 teacher 或 ground-truth 生成一批固定数据，然后让 student 在这些固定轨迹上模仿 teacher：

$$
y \sim \pi_{\mathrm{teacher}}(\cdot \mid c, x)
$$

$$
D_{\mathrm{KL}}
\left(
\pi_{\mathrm{teacher}}(\cdot \mid c, x, y_{<t})
\|
\pi_\theta(\cdot \mid x, y_{<t})
\right)
$$

这有两个问题。

第一是 **exposure bias**。训练时 student 看到的是 teacher-generated 或 ground-truth prefix；推理时 student 必须根据自己生成的 prefix 继续写：

$$
y \sim \pi_\theta(\cdot \mid x)
$$

如果 student 早期生成偏了，后续会进入 teacher 固定数据里没有覆盖的状态。off-policy 训练并没有告诉 student：在自己犯错后、在自己实际访问到的 prefix 上，应该怎样使用已经蒸馏进去的 context knowledge 来纠正。

第二是 **forward KL 的 mode-covering 倾向**。Forward KL 会要求 student 覆盖 teacher 分布中所有有概率的 token。当 teacher 的 context-aware 分布很复杂，而 student 容量或状态又跟不上时，student 可能学成过宽的分布，论文把这和 hallucinations / overly broad distributions 联系起来。

OPCD 的核心改动是把 context distillation 改成 **on-policy**。训练时先让 student 在没有 context 的情况下生成自己的轨迹：

$$
y \sim \pi_\theta(\cdot \mid x)
$$

然后在 student 自己访问到的每个 prefix 上，让 teacher 额外看到 context `c`，计算 context-conditioned teacher distribution：

$$
\pi_{\mathrm{teacher}}(\cdot \mid c, x, y_{<t})
$$

训练 loss 是 student 分布到 context-conditioned teacher 分布的 reverse KL：

$$
L(\theta)
=
\mathbb{E}_{(x,c)\sim D,\; y\sim \pi_\theta(\cdot|x)}
\left[
\frac{1}{|y|}
\sum_{t=1}^{|y|}
D_{\mathrm{KL}}
\left(
\pi_\theta(\cdot \mid x, y_{<t})
\|
\pi_{\mathrm{teacher}}(\cdot \mid c, x, y_{<t})
\right)
\right]
$$

逐 token 写成：

$$
D_{\mathrm{KL}}
\left(
\pi_\theta(\cdot \mid x, y_{<t})
\|
\pi_{\mathrm{teacher}}(\cdot \mid c, x, y_{<t})
\right)
=
\sum_{y'_t\in V}
\pi_\theta(y'_t \mid x, y_{<t})
\left[
\log \pi_\theta(y'_t \mid x, y_{<t})
-
\log \pi_{\mathrm{teacher}}(y'_t \mid c, x, y_{<t})
\right]
$$

论文实现中用 student top-`k` token 近似这个 KL。

所以那句话的意思是：

```text
off-policy context distillation:
  student 在 teacher/固定轨迹上学 context knowledge
  但推理时 student 走自己的轨迹，可能不知道在自己的状态上怎么用这些知识

OPCD:
  student 先走自己的轨迹
  teacher 在这些 student-visited states 上带着 context 给监督
  student 因此学会在自己真实会遇到的状态上使用并内化 context knowledge
```

论文中的两个主要应用正好对应这句话里的 context 来源：

1. **Experiential knowledge distillation**

   模型从历史解题轨迹中提取 transferable experiential knowledge，把它累计成 context `c`，再用 OPCD 蒸馏进参数。推理时不需要再把长历史 trace 放进上下文。

2. **System prompt distillation**

   把 medical / safety 等任务上的 optimized system prompt 当作 context `c`，teacher 看 system prompt，student 不看 system prompt，但通过 OPCD 学会 system prompt 所诱导的行为。

一句话总结：这篇论文认为 context distillation 的关键不是只在固定 teacher 数据上模仿“带 context 的输出”，而是要在 **student 自己会生成的轨迹** 上，让 **带 context 的 teacher** 告诉它每一步应该怎样调整；这样 student 才更可能把 prompt、历史经验或 system prompt 中的知识真正内化到参数里，并在推理时不用 context 也能使用这些知识。

## 2602.22495 - Reinforcement-aware Knowledge Distillation for LLM Reasoning

本地文件：`../papers/llm_opd_methods/2026_ucla_reinforcement_aware_knowledge_distillation_for_llm_reasoning_2602.22495.pdf`

### Q: “把 KD 和 RL 直接结合时容易出现 distribution mismatch 与 objective interference：teacher KL regularizer 会和 reward maximization 抢目标，需要手动 loss balancing”是什么意思？

这篇论文研究的是：在 reasoning RL post-training 中，怎样把大 teacher 的知识注入小 student，而不是只做普通 RL 或离线 SFT 蒸馏。

最直接的做法有两类。

第一类是 **offline distillation**：先让 teacher 生成 reasoning traces，再用 SFT 训练 student：

$$
y \sim \pi_T(\cdot \mid x)
$$

$$
\max_{\theta_S}
\log \pi_{\theta_S}(y \mid x)
$$

问题是这些数据来自 teacher distribution，而不是 student 自己 RL 时会访问的 distribution。student 推理或 RL 训练时会走自己的轨迹：

$$
y \sim \pi_{\theta_S}(\cdot \mid x)
$$

这就是论文说的 **distribution mismatch**：teacher traces / fixed dataset 上的监督，不一定覆盖 student 自己生成错误、偏离或探索到的状态。

第二类是把 KD 直接加到 RL objective 里。以 GRPO 为例，student 根据 reward 最大化：

$$
J_{\mathrm{GRPO}}(\theta_S)
=
\mathbb{E}
\left[
\min
\left(
r_{i,t}(\theta_S)\hat A_{i,t},
\mathrm{clip}(r_{i,t}(\theta_S),1-\epsilon,1+\epsilon)\hat A_{i,t}
\right)
\right]
-
\beta D_{\mathrm{KL}}(\pi_{\theta_S}\|\pi_{\mathrm{ref}})
$$

其中：

$$
r_{i,t}(\theta_S)
=
\frac{
\pi_{\theta_S}(y^{(i)}_t \mid x,y^{(i)}_{<t})
}{
\pi_{\theta_{S,\mathrm{old}}}(y^{(i)}_t \mid x,y^{(i)}_{<t})
}
$$

朴素 KDRL 会额外加一个 teacher KL regularizer，或者把 reference 换成 teacher：

$$
D_{\mathrm{KL}}(\pi_{\theta_S}\|\pi_{\theta_T})
=
\mathbb{E}_{y_t\sim \pi_{\theta_S}}
\left[
\log \pi_{\theta_S}(y_t\mid x,y_{<t})
-
\log \pi_{\theta_T}(y_t\mid x,y_{<t})
\right]
$$

这个 KL 本质上就是 reverse KL distillation：让 student 靠近 teacher。

问题在于这个 teacher KL 是一个 **固定的、和 advantage 无关的约束**。它会在所有 token / states 上都推动 student 模仿 teacher，不管当前 reward signal 是否支持这种模仿。

这就是论文说的 **objective interference**：

```text
RL reward maximization:
  增加高 advantage 行为的概率，降低低 advantage 行为的概率

teacher KL regularizer:
  不管 advantage，统一把 student 拉向 teacher

二者可能方向一致，也可能方向冲突
```

例如：

- 某个 token / trajectory 在当前 task reward 下 advantage 很高，但 teacher 概率低：RL 想增加它，teacher KL 想压它。
- 某个 token / trajectory advantage 很低，但 teacher 概率高：RL 想降低它，teacher KL 想保留它。
- 早期 student 和 teacher 差距大时，KL 项可能很大，产生 outlier gradients 或不稳定训练。

因此普通 KDRL 需要手动调 loss balancing：

$$
J
=
J_{\mathrm{RL}}
-
\lambda_{\mathrm{KD}}
D_{\mathrm{KL}}(\pi_{\theta_S}\|\pi_{\theta_T})
$$

如果 `lambda_KD` 太大，训练主要变成 imitation，reward optimization 被压住；如果太小，teacher signal 又几乎不起作用。论文实验中也提到 explicit-KL-style distillation 对 reward maximization 和 auxiliary KL regularizer 的耦合很敏感。

这篇论文提出 **RL-aware distillation (RLAD)**，核心原则是 **selective imitation**：只有当 teacher guidance 对当前 reward-driven update 有帮助时，才更强地模仿 teacher。

具体算法组件是 **Trust Region Ratio Distillation (TRRD)**。它不把 teacher 当作独立 KL penalty，而是把 teacher 放进 GRPO/PPO 风格的 likelihood ratio 里：

$$
r^{\mathrm{TRRD}}_{i,t}(\theta_S)
=
\left(
\frac{
\pi_{\theta_S}(y^{(i)}_t \mid x,y^{(i)}_{<t})
}{
\pi_{\theta_{S,\mathrm{old}}}(y^{(i)}_t \mid x,y^{(i)}_{<t})
}
\right)^\alpha
\left(
\frac{
\pi_{\theta_S}(y^{(i)}_t \mid x,y^{(i)}_{<t})
}{
\pi_{\theta_T}(y^{(i)}_t \mid x,y^{(i)}_{<t})
}
\right)^{1-\alpha}
$$

然后用这个 ratio 替换 GRPO 里的普通 ratio：

$$
J_{\mathrm{RLAD}}(\theta_S)
=
\mathbb{E}
\left[
\min
\left(
r^{\mathrm{TRRD}}_{i,t}(\theta_S)\hat A_{i,t},
\mathrm{clip}(r^{\mathrm{TRRD}}_{i,t}(\theta_S),1-\epsilon,1+\epsilon)\hat A_{i,t}
\right)
\right]
-
\beta D_{\mathrm{KL}}(\pi_{\theta_S}\|\pi_{\mathrm{ref}})
$$

TRRD 可以理解为把 trust region 的锚点从单纯 old student policy，换成 old student 和 teacher 的 mixture anchor：

$$
\pi_{\mathrm{mix}}
\propto
\pi_{\theta_{S,\mathrm{old}}}^{\alpha}
\pi_{\theta_T}^{1-\alpha}
$$

所以：

$$
r^{\mathrm{TRRD}}
=
\frac{\pi_{\theta_S}}{\pi_{\mathrm{mix}}}
$$

论文也把它解释成一种隐式加权 KL：

$$
\mathbb{E}[R(x,y)]
-
\alpha D_{\mathrm{KL}}(\pi_{\theta_S}\|\pi_{\theta_{S,\mathrm{old}}})
-
(1-\alpha)D_{\mathrm{KL}}(\pi_{\theta_S}\|\pi_{\theta_T})
$$

但关键区别是：这个 teacher term 被嵌入 advantage-weighted clipped policy update 中，而不是作为一个独立、固定强度的 KL loss。

它为什么能缓解 objective interference？

- 当 advantage 为正时，RL 想提高该 token 概率；如果 teacher 也给高概率，TRRD 的 clipping 会允许 student 更充分地增加它。
- 当 advantage 为正但 teacher 给低概率时，更新更早被 trust region 限制，避免盲目偏离 teacher。
- 当 advantage 为负时，RL 想降低该 token 概率；如果 teacher 给高概率，TRRD 会阻止 student 过度远离 teacher。
- 当 advantage 接近 0 时，teacher influence 也基本消失，因为 advantage-weighted update 本身没有强学习信号。

也就是说，teacher signal 不再和 reward maximization 平行抢目标，而是被 reward / advantage 调制：

```text
普通 KDRL:
  reward loss + teacher KL loss
  需要手动调二者权重

RLAD/TRRD:
  teacher 进入 advantage-weighted trust-region ratio
  teacher imitation 随 advantage 和 clipping 自动起作用
```

论文中的 “reinforcement-aware” 就是这个意思：teacher guidance 不是无条件模仿，而是感知当前 RL update 的 advantage 和 trust region，只在与 reward-driven learning 相容的地方更强地发挥作用。

一句话总结：这篇论文认为朴素 KD+RL 会把“拿高 reward”和“像 teacher”写成两个可能冲突的目标；RLAD/TRRD 则把 teacher 放进 GRPO/PPO 的 ratio 与 trust region 里，让 teacher imitation 受 advantage 调制，从而减少 distribution mismatch 和 objective interference，并降低手动调 KD/RL loss balance 的需求。

## 2603.07079 - Entropy-Aware On-Policy Distillation of Language Models

本地文件：`../papers/llm_opd_methods/2026_kaist_entropy_aware_on_policy_distillation_of_language_models_2603.07079.pdf`

### Q: “标准 reverse KL 的 mode-seeking 性质在 teacher entropy 高时会压缩多样性、产生不稳定 learning signal”是什么意思？为什么是在 teacher entropy 高时压缩多样性？

这篇论文分析的是标准 OPD 里常用的 **reverse KL** 目标为什么会在某些 token 位置上失败。

在 token-level distillation 中，teacher 和 student 分别给出：

$$
\pi_{\mathrm{te}}(\cdot \mid c_t),
\qquad
\pi_\theta(\cdot \mid c_t)
$$

其中 `c_t` 是当前 prompt 加上已经生成的 prefix。

**Forward KL** 是 teacher 到 student：

$$
D_{\mathrm{KL}}
\left(
\pi_{\mathrm{te}}(\cdot \mid c_t)
\|
\pi_\theta(\cdot \mid c_t)
\right)
=
\mathbb{E}_{x\sim\pi_{\mathrm{te}}}
\left[
\log
\frac{
\pi_{\mathrm{te}}(x\mid c_t)
}{
\pi_\theta(x\mid c_t)
}
\right]
$$

它会惩罚 student 没有覆盖 teacher 认为可能的 token，因此是 **mode-covering**。

**Reverse KL** 是 student 到 teacher：

$$
D_{\mathrm{KL}}
\left(
\pi_\theta(\cdot \mid c_t)
\|
\pi_{\mathrm{te}}(\cdot \mid c_t)
\right)
=
\mathbb{E}_{x\sim\pi_\theta}
\left[
\log
\frac{
\pi_\theta(x\mid c_t)
}{
\pi_{\mathrm{te}}(x\mid c_t)
}
\right]
$$

它的期望在 student 分布上，所以 student 只会为自己采到、自己给概率的 token 付代价。如果 teacher 有多个合理模式，但 student 当前只覆盖其中一个，reverse KL 不会强迫 student 去覆盖它没访问到的其它 teacher modes。这就是 **mode-seeking**。

在 OPD 中，标准 reverse-KL reward 可以写成：

$$
A_t
=
\log \pi_{\mathrm{te}}(x_t\mid c_t)
-
\log \pi_\theta(x_t\mid c_t)
$$

student 在自己 rollout 的 token 上，根据 teacher 是否比 student 更偏好该 token 来更新。

为什么问题主要出现在 **teacher entropy 高** 的位置？

teacher token entropy 是：

$$
H^{\mathrm{te}}_t
=
-
\sum_{x\in V}
\pi_{\mathrm{te}}(x\mid c_t)
\log \pi_{\mathrm{te}}(x\mid c_t)
$$

当 teacher entropy 低时，teacher 分布很尖：

```text
token A: 0.90
token B: 0.03
token C: 0.02
...
```

这说明 teacher 很确定下一步应该是什么。此时 reverse KL 的 mode-seeking 性质反而合适，因为 teacher 本来就只有一个主模式。student 追这个主模式，不会损失太多真实多样性。

当 teacher entropy 高时，teacher 分布更平：

```text
token A: 0.25
token B: 0.22
token C: 0.18
token D: 0.15
...
```

这通常表示当前位置存在多个合理 continuation。论文特别强调，在数学和多步 reasoning 中，高熵 token 不只是噪声，而可能代表：

- 多条可行推理路径
- 合理的表达变体
- teacher 的真实不确定性
- 对后续探索有用的多样性

但 reverse KL 的期望来自 student。如果 student 当前只采到 token A，它主要会围绕 A 更新；teacher 对 B/C/D 也给了不低概率，但 student 没采到它们，就几乎没有梯度去恢复这些模式。久而久之，student 会把一个本来多峰的 teacher 分布压成更窄的分布：

```text
teacher high-entropy:
  A/B/C/D 都合理

student reverse-KL OPD:
  主要押 A
  B/C/D 被低估甚至消失
```

这就是“teacher entropy 高时压缩多样性”的原因：**高熵意味着 teacher 的概率质量分散在多个合理 token 上，而 reverse KL 不会主动覆盖 student 没访问到的 teacher modes，因此更容易把这种分散结构压成少数 dominant modes。**

论文用实验验证了这一点。它比较 Qwen3-8B teacher 和 Qwen3-1.7B-Base OPD student 的 token entropy histogram，发现标准 reverse-KL OPD 后，student 保留的高熵 token 明显少于 teacher：

```text
teacher: 18.5% high-entropy tokens
reverse-KL OPD student: 6.8% high-entropy tokens
```

这说明 student 的分布比 teacher 更尖、更窄，多样性下降。

为什么还会有 **unstable learning signal**？

在高熵位置，teacher 没有一个压倒性正确的 top-1 token，多个 token 概率接近。此时 student 采样稍微变动，就可能采到不同 token；由于 reverse-KL reward 是单样本式地看：

$$
\log \pi_{\mathrm{te}}(x_t\mid c_t)
-
\log \pi_\theta(x_t\mid c_t)
$$

不同 sampled token 的 reward 差异会导致更新方向频繁变化。论文的 toy experiment 中，高熵 teacher 下 student 的 top-1 index 经常改变，top-10 set 也难以收敛；低熵 teacher 下则稳定得多。这说明当 teacher 不确定时，reverse-KL reward 给出的 token-level learning signal 更容易抖动。

论文提出 **Entropy-Aware On-Policy Distillation (EOPD)**。核心思想是：

```text
低 teacher entropy:
  teacher 很确定
  用 reverse KL，高效追主模式

高 teacher entropy:
  teacher 有多个合理 continuation
  加 forward KL，帮助 student 覆盖 teacher 的分布结构
```

EOPD 的 token-level loss 是：

$$
L^{\mathrm{EOPD}}_t(\theta;c_t)
=
L^{\mathrm{OPD}}_t(\theta;c_t)
+
\alpha\cdot \mathbf{1}[H^{\mathrm{te}}_t>\tau]
L^{\mathrm{FKL}}_t(\theta;c_t)
$$

其中：

$$
L^{\mathrm{FKL}}_t(\theta;c_t)
=
D_{\mathrm{KL}}
\left(
\pi_{\mathrm{te}}(\cdot\mid c_t)
\|
\pi_\theta(\cdot\mid c_t)
\right)
$$

`tau` 是 teacher entropy threshold。只有当：

$$
H^{\mathrm{te}}_t>\tau
$$

才额外启用 forward KL。

实际实现中，forward KL 不在全词表上算，而是在 teacher top-`k` tokens 上近似：

$$
L^{\mathrm{FKL}}_t(\theta;c_t)
\approx
\sum_{x\in S^k_t}
\tilde{\pi}_{\mathrm{te}}(x\mid c_t)
\log
\frac{
\tilde{\pi}_{\mathrm{te}}(x\mid c_t)
}{
\pi_\theta(x\mid c_t)
}
$$

这样既避免 full forward KL 的内存开销，也避免让 student 学 teacher 的低概率长尾。

一句话总结：这篇论文的意思是，reverse KL 适合 teacher 很确定的 token，因为只追主模式就够了；但当 teacher entropy 高时，teacher 的知识本身分布在多个合理 continuation 上，reverse KL 只根据 student 自己采到的少数 token 更新，会忽略未访问的 teacher modes，从而压缩多样性并造成学习信号抖动。EOPD 用 teacher entropy 判断何时加入 forward KL，在高不确定位置显式保留 teacher 的多峰分布。

## 2603.11137 - Scaling Reasoning Efficiently via Relaxed On-Policy Distillation

本地文件：`../papers/llm_opd_methods/2026_academic_scaling_reasoning_efficiently_via_relaxed_on_policy_distilla_2603.11137.pdf`

### Q: “OPD 在 reasoning transfer 中容易不稳定和 negative transfer，原因是标准 OPD 对 teacher 的 strict imitation 太硬”是什么意思？

这篇论文讨论的是 reasoning model 的 on-policy distillation。标准 OPD/RKL 的形式是：student 用自己的 policy 生成轨迹，再让 student 在这些轨迹上贴近 teacher。

对于 prompt `q` 和 student 生成的 response `o`，reverse KL 可以写成：

$$
D_{\mathrm{KL}}(\pi_\theta\|\pi_T)
=
-
\mathbb{E}_{q\sim Q,\;o\sim\pi_\theta(\cdot|q)}
\left[
\log
\frac{\pi_T(o|q)}{\pi_\theta(o|q)}
\right]
$$

也就是最大化 teacher-student log-likelihood ratio：

$$
\log
\frac{\pi_T(o|q)}{\pi_\theta(o|q)}
$$

论文把这个目标改写成 token-level policy optimization。对第 `i` 条 response 的第 `t` 个 token：

$$
R_{i,t}(\theta)
=
\log
\frac{
\pi_T(o_{i,t}\mid q,o_{i,<t})
}{
\pi_\theta(o_{i,t}\mid q,o_{i,<t})
}
$$

这个 `R_{i,t}` 可以看成 teacher 给 student 这个 token 的 reward：

- 如果 teacher 比 student 更喜欢这个 token，`R_{i,t}>0`
- 如果 teacher 比 student 更不喜欢这个 token，`R_{i,t}<0`
- 如果 teacher 和 student 已经接近，`R_{i,t}\approx 0`

带 importance ratio 的 RKL/OPD 目标可写成：

$$
J_{\mathrm{RKL}}(\theta)
=
\mathbb{E}
\left[
\frac{1}{\sum_i |o_i|}
\sum_i \sum_t
\rho_{i,t}(\theta) R_{i,t}(\theta)
\right]
$$

其中：

$$
\rho_{i,t}(\theta)
=
\frac{
\pi_\theta(o_{i,t}\mid q,o_{i,<t})
}{
\pi_{\theta_{\mathrm{old}}}(o_{i,t}\mid q,o_{i,<t})
}
$$

论文的一个关键观察是：OPD 可以看作 policy-gradient optimization，只不过 reward 来自 teacher-student log-ratio，而不是外部 verifier：

$$
\nabla_\theta J_{\mathrm{RKL}}
=
\mathbb{E}
\left[
\rho_{i,t}(\theta)
R_{i,t}(\theta)
\nabla_\theta
\log \pi_\theta(o_{i,t}\mid q,o_{i,<t})
\right]
$$

如果对 `R_{i,t}` 做 stop-gradient，把它当固定 reward，理论期望梯度和原始 RKL 一致，但有限样本训练会更稳，因为去掉了 `R_{i,t}` 自身随 `theta` 变化带来的高方差项。

那么“strict imitation 太硬”具体是什么意思？

标准 OPD/RKL 会把 teacher 的偏好当成强约束：student 采到的每个 token 都要按 teacher-student log-ratio 被奖励或惩罚。这个机制在 reasoning transfer 中有三个问题。

第一，**teacher 给极低概率的 student token 会产生 heavy-tailed negative rewards**。

如果 student 探索时采到了 teacher 几乎不给概率的 token：

$$
\pi_T(o_{i,t}\mid q,o_{i,<t}) \to 0
$$

那么：

$$
R_{i,t}
=
\log
\frac{\pi_T(o_{i,t}\mid q,o_{i,<t})}
{\pi_\theta(o_{i,t}\mid q,o_{i,<t})}
\to
-\infty
$$

这会产生非常大的负 reward，主导梯度，导致 destructive updates。也就是说，student 只要走出 teacher 高概率区域一点点，就会被非常强地惩罚。

这就是“strict imitation”：不是温和地利用 teacher guidance，而是把偏离 teacher 的 token 当成严重错误压下去。

第二，**大量 token 的 reward 接近 0，导致信号低效**。

论文观察到，多数低熵、确定性 token 上 teacher 和 student 已经很一致：

$$
R_{i,t}\approx 0
$$

这些 token 消耗 batch 和显存，但几乎不给有效学习信号。reasoning transfer 真正有信息量的往往是高熵分叉点，也就是推理路径开始分化的位置。

第三，**strict negative punishment 会造成 entropy collapse，进而 negative transfer**。

reasoning 需要探索多条可能的 solution paths。但标准 RKL 会快速降低 policy entropy：student 越来越只保留少数 teacher 高概率模式。这样虽然看起来更像 teacher，但会损失探索能力。

论文还指出，单纯提高 sampling temperature 也不解决问题。更高温度会让 student 采到更多偏离 teacher 的 token，这些 token 又触发更大的负 reward，进一步放大 reward variance。

所以出现一个冲突：

```text
reasoning transfer 需要探索多样路径
standard OPD 又强烈惩罚偏离 teacher 的探索 token
```

这就是 negative transfer 的来源之一：训练后 student 不仅没有稳定提升，甚至可能低于初始化 base model。论文在 7B student 上观察到 vanilla RKL 不稳定，而 REOPOLD 能稳定提升。

论文提出 **REOPOLD (Relaxed On-Policy Distillation)**，核心就是放松这种硬模仿约束。它不是取消 teacher，而是“temperately and selectively” 使用 teacher reward。

REOPOLD 的目标是：

$$
J_{\mathrm{REOPOLD}}(\theta)
=
\mathbb{E}
\left[
\frac{
1
}{
\sum_i\sum_t M^{(k)}_{i,t}
}
\sum_i\sum_t
\rho_{i,t}(\theta)
\hat R^\lambda_{i,t}(\theta)
M^{(k)}_{i,t}
\right]
$$

它有三个关键设计。

**1. Mixture-based reward clipping**

把原始 reward：

$$
R_{i,t}
=
\log
\frac{\pi_T}{\pi_\theta}
$$

裁剪成：

$$
\hat R^\lambda_{i,t}(\theta)
=
\max
\left(
\mathrm{sg}(R_{i,t}(\theta)),
\log\frac{\lambda}{1-\lambda}
\right)
$$

这样当 teacher 给某 token 极低概率时，负 reward 不会继续趋向 `-\infty`。它直接缓解 heavy-tailed negative rewards 和训练不稳定。

**2. Entropy-guided token-level dynamic sampling**

论文发现低熵 token 上 reward 大多接近 0，高熵 token 更可能是 teacher/student 分歧和 reasoning 分叉点。因此 refinement 阶段只在高熵 token 上计算梯度：

$$
M_{i,t}
=
\mathbf{1}[H^i_t\ge \tau_\beta]
$$

其中 `H^i_t` 是 student policy 在该 token 位置的 entropy，`\tau_\beta` 是 batch 内 top-`beta` percentile threshold。

这相当于把训练预算集中在信息量大的 token 上，而不是让海量低信号 token 稀释梯度。

**3. Exploration-to-refinement multi-stage training**

训练分两阶段。

探索阶段：

$$
M_{i,t}
=
\mathbf{1}
\left[
R_{i,t}(\theta)
\ge
\log\frac{\lambda}{1-\lambda}
\right]
$$

这个阶段过滤掉特别强的负 reward，避免一开始就把 student 的探索空间压扁。论文说它类似 SFT：强化正向行为，但不过度惩罚 exploration errors。

精炼阶段：

$$
M_{i,t}
=
\mathbf{1}[H^i_t\ge \tau_\beta]
$$

这个阶段重新引入负反馈，但只聚焦在高熵、真正分歧大的 token 上，让模型收敛到更高质量的 reasoning paths。

所以“Relaxed OPD”的 relaxed 不是说 teacher supervision 变弱到无用，而是：

```text
不再对所有 token 无条件严格模仿 teacher
不再让极端负 reward 主导训练
不再把训练预算浪费在 near-zero reward tokens
先保留探索，再聚焦精炼
```

一句话总结：这篇论文认为标准 OPD 在 reasoning transfer 中的问题，是把 teacher-student log-ratio 当成过硬的逐 token 模仿约束；当 student 探索到 teacher 低概率 token 时，会产生极端负 reward，导致训练不稳定、entropy collapse 和 negative transfer。REOPOLD 把 OPD 重写成 policy optimization 后，通过 reward clipping、entropy-based token sampling 和 exploration-to-refinement 阶段训练，放松 teacher imitation，使 teacher 信号更选择性、更稳定地用于 reasoning transfer。

## 2601.07155 vs 2603.07079 - Veto 和 EOPD 的区别

本地文件：

- `../papers/llm_opd_methods/2026_kaist_stable_on_policy_distillation_through_adaptive_target_reform_2601.07155.pdf`
- `../papers/llm_opd_methods/2026_kaist_entropy_aware_on_policy_distillation_of_language_models_2603.07079.pdf`

### Q: 这两篇论文的区别是什么？不都是动态调整 FKL 和 RKL 的比例吗？

不是。两篇都在处理 OPD 里 KL 方向带来的问题，但它们动态调整的对象不同。

一句话区分：

```text
2601.07155 Veto:
  动态重构 target distribution，让 teacher target 不那么硬
  重点解决 teacher-student gap 下的坏梯度和 mode collapse

2603.07079 EOPD:
  按 teacher entropy 在部分 token 上额外加入 forward KL
  重点解决高 teacher-entropy 位置的多样性压缩
```

`2601.07155` 的核心不是“动态调整 FKL/RKL 比例”，而是构造一个中间目标分布：

$$
Q_\beta(\cdot \mid x,y_{<t})
\propto
\exp(z_T+\beta z_S)
=
\frac{
P_T(\cdot \mid x,y_{<t})
P_S(\cdot \mid x,y_{<t})^\beta
}{Z}
$$

其中：

- `P_T` 是 teacher distribution
- `P_S` 是 student 当前 distribution
- `beta` 控制 student 当前信念对 target 的影响

然后可以用：

$$
D_{\mathrm{KL}}(Q_\beta\|P_S)
$$

也可以用：

$$
D_{\mathrm{KL}}(P_S\|Q_\beta)
$$

关键点是：Veto 先把 teacher target `P_T` 改造成 `Q_beta`，再做 KL。它没有根据 token entropy 在 FKL 和 RKL 之间路由，也不是把 FKL/RKL 做成一个 token-wise mixture。

`beta` 的作用是调 target 的硬度：

- 在 forward KL regime 中，如果 teacher 喜欢某 token 但 student 当前几乎不给概率，`P_S^\beta` 会压低该 token 在 `Q_beta` 中的权重，避免 forward KL 对 student 的 ignorant tokens 产生 pathological gradients。
- 在 reverse KL regime 中，`beta` 改变 student 的 entropy regularization 效果，像 decisiveness knob：控制模式集中和多样性之间的 trade-off。

也就是说，Veto 的动态性来自：

$$
Q_\beta = f(P_T,P_S,\beta)
$$

它关注的是 **teacher-student gap**：当 teacher 和 student 差太远时，直接模仿 teacher target 太硬。

`2603.07079` 的 EOPD 则是真正显式地做 KL 方向的 token 条件选择。它保留标准 OPD/RKL，并在 teacher entropy 高的位置额外加 forward KL：

$$
L^{\mathrm{EOPD}}_t
=
L^{\mathrm{OPD}}_t
+
\alpha\cdot
\mathbf{1}[H^{\mathrm{te}}_t>\tau]
L^{\mathrm{FKL}}_t
$$

其中：

$$
H^{\mathrm{te}}_t
=
-
\sum_{x\in V}
\pi_{\mathrm{te}}(x\mid c_t)
\log \pi_{\mathrm{te}}(x\mid c_t)
$$

当 teacher entropy 低时：

$$
H^{\mathrm{te}}_t\le \tau
$$

EOPD 退化成标准 reverse-KL OPD：

$$
L^{\mathrm{EOPD}}_t=L^{\mathrm{OPD}}_t
$$

当 teacher entropy 高时：

$$
H^{\mathrm{te}}_t>\tau
$$

EOPD 加上 forward KL：

$$
L^{\mathrm{FKL}}_t
=
D_{\mathrm{KL}}
\left(
\pi_{\mathrm{te}}(\cdot\mid c_t)
\|
\pi_\theta(\cdot\mid c_t)
\right)
$$

这样做是为了覆盖 teacher 在高不确定位置的多个合理 continuation，避免 reverse KL 只追一个 mode。

所以 EOPD 的动态性来自：

$$
\mathbf{1}[H^{\mathrm{te}}_t>\tau]
$$

它关注的是 **teacher uncertainty**：当 teacher 自己也不确定、分布高熵时，reverse KL 的 mode-seeking 会把多峰分布压窄。

两篇论文的区别可以压缩成下表：

| 维度 | `2601.07155` Veto | `2603.07079` EOPD |
|---|---|---|
| 主要 failure mode | teacher-student gap 大时，FKL 梯度爆炸；RKL mode collapse | teacher entropy 高时，RKL 压缩多样性并产生不稳定信号 |
| 机制归因 | teacher target 太硬，student 低置信 token 上监督过强 | teacher 高熵表示多个合理模式，RKL 不覆盖 student 未访问的 teacher modes |
| 动态变量 | student 当前分布 `P_S` 和 `beta` | teacher token entropy `H_te` 和阈值 `tau` |
| 改动对象 | target distribution：`P_T -> Q_beta` | loss composition：高熵 token 上 `RKL + FKL` |
| 是否动态调 FKL/RKL 比例 | 不是。它可用于 FKL 或 RKL，但核心是 target reformulation | 是，更接近 token-wise 条件加 FKL |
| 主要目标 | 稳定 teacher-student gap 下的 OPD | 保留 teacher 高不确定位置的多样性 |
| 直觉 | “teacher 说得太硬，先让 target 看看 student 当前会不会” | “teacher 自己也不确定时，不要只学一个 mode” |

二者看起来相似，是因为都讨论了 FKL/RKL 的典型问题：

- FKL：mode-covering，但 teacher-student gap 大时可能梯度很大
- RKL：mode-seeking，但可能 diversity collapse

但解决路径不同：

```text
Veto:
  改 teacher target
  让 KL 的目标变软、变得和 student 当前能力更匹配

EOPD:
  保留 RKL 的高效主模式学习
  只在 teacher 高熵位置补 FKL 来覆盖多模式
```

可以这样理解两篇论文的适用场景：

- 如果问题是 **student 太弱，teacher target 太远，直接 KL 训练不稳定**，更像 Veto 的问题。
- 如果问题是 **teacher 在某些 token 上本来就有多种合理选择，RKL 把这些选择压成单一路径**，更像 EOPD 的问题。

一句话总结：两篇论文都在修 OPD 的 KL 问题，但 `2601.07155` 是 **adaptive target reformulation**，用 `Q_beta` 软化 teacher target；`2603.07079` 是 **entropy-aware KL routing/augmentation**，在高 teacher entropy token 上额外加 forward KL。前者主要面向 teacher-student gap，后者主要面向 teacher uncertainty。

## Sampled-Token OPD vs Full/Top-k Distribution OPD

### Q: 目前 `问答记录.md` 中提到的这些论文都是使用 sampled-token OPD 吗？

不是。这里要区分两个层面：

1. **rollout 是否 on-policy**

   也就是 response / trajectory 是否由 student 当前 policy 生成：

   $$
   y \sim \pi_\theta(\cdot\mid x)
   $$

2. **每个 token 位置的 distillation loss 怎么算**

   是只看 sampled token：

   $$
   x_t \sim \pi_\theta(\cdot\mid c_t)
   $$

   还是在该位置上计算 vocabulary / top-`k` 分布 KL：

   $$
   D_{\mathrm{KL}}
   \left(
   p_T(\cdot\mid c_t)\|p_S(\cdot\mid c_t)
   \right)
   $$

很多论文是 **on-policy rollout**，但不是 **sampled-token loss**。

当前 `问答记录.md` 中涉及的论文大致可以分为：

| 论文 | 是否 student on-policy rollout | token-level loss 形态 | 是否 sampled-token OPD |
|---|---:|---|---:|
| `2601.07155` Veto | 是 | 构造 `Q_beta` 后做 KL；论文是 objective-level reformulation，可用于 FKL/RKL | 不主要是 sampled-token；更接近分布级 target reformulation |
| `2601.18734` OPSD / Self-Distilled Reasoner | 是 | full-vocabulary logit distillation，比较 `p_T` 和 `p_S`；也讨论 sampled-token reward 视角 | 主方法不是 sampled-token，是 full-vocabulary |
| `2601.20802` SDPO | 是 | feedback-conditioned self-teacher 与 student 的 logit-level KL；为省显存用 student top-`K` distillation | 不是单 sampled-token，主要是 top-`K` 分布近似 |
| `2602.04942` PI-Distill / OPSD | `pi`-Distill 用 teacher rollout；OPSD 用 student rollout | reverse KL penalty / GRPO-style trajectory objective；算法里多写成 trajectory-level KL reward | OPSD 部分可看作 sampled trajectory 上的 KL reward，但不是典型 full-vocab OPD 论文 |
| `2602.12125` G-OPD / ExOPD | 理论上分析 OPD | 把 OPD 等价成 dense KL-constrained RL，引入 reward scaling/reference | 是理论推广，不应简单归为 sampled-token |
| `2602.12275` OPCD | 是 | student rollout 上，对 context-conditioned teacher 做 reverse KL；实现用 student top-`k` token 近似 analytic KL | 不是单 sampled-token，是 top-`k` distribution KL |
| `2602.22495` RLAD / TRRD | 是 | PPO/GRPO-style likelihood ratio，teacher-old-policy mixture anchor，advantage-weighted | 是 sampled-token / sampled-trajectory ratio 型 |
| `2603.07079` EOPD | 是 | RKL 用 sampled-token reward/OPD；高 teacher entropy 位置额外加 teacher top-`k` forward KL | 混合：基础 RKL 是 sampled-token，FKL 补项是 top-`k` 分布 |
| `2603.11137` REOPOLD | 是 | teacher-student log-likelihood ratio 作为 token reward，带 stop-gradient、reward clipping、mask | 是 sampled-token ratio/reward 型 |

所以更准确的说法是：

```text
这些论文大多都是 on-policy rollout：
  student 生成自己的轨迹，再让 teacher/conditioned teacher 在这些 states 上提供信号。

但它们不都使用 sampled-token loss：
  有的是 full-vocabulary KL
  有的是 top-k KL
  有的是 sampled-token log-ratio reward
  有的是理论上重写 OPD objective
```

可以按实现成本理解：

**Full-vocabulary KL**

在每个位置存 teacher 和 student 的完整 vocab logits：

$$
D_{\mathrm{KL}}(p_T(\cdot\mid c_t)\|p_S(\cdot\mid c_t))
$$

信号最密，但显存/通信开销最大。`2601.18734` 的 OPSD 主实验就是这种 full-vocabulary logit distillation。

**Top-`k` distribution KL**

只保留 top-`k` token：

$$
\sum_{x\in S^k_t}
\tilde p_T(x\mid c_t)
\log
\frac{\tilde p_T(x\mid c_t)}{p_S(x\mid c_t)}
$$

这是 full KL 的近似，保留一部分分布信息，降低显存。`2601.20802`、`2602.12275`、`2603.07079` 的 FKL 补项都属于这个方向。

**Sampled-token log-ratio / reward**

只对实际采到的 token 算：

$$
R_t
=
\log p_T(x_t\mid c_t)
-
\log p_S(x_t\mid c_t)
$$

或者放进 PPO/GRPO ratio。显存低、适合长 reasoning，但信号更 noisy，也更容易出现 high-variance / negative-reward 问题。`2603.11137` REOPOLD 和 `2602.22495` RLAD/TRRD 更接近这一类。

一句话总结：`问答记录.md` 目前提到的论文大多是 **student-rollout on-policy**，但不都是 **sampled-token OPD**；sampled-token 只是其中一类低显存、RL-style 的实现形态，和 full-vocabulary/top-`k` distribution KL 并列。

## 2604.03128 - Self-Distilled RLVR 文档表述审核

本地文件：`../papers/llm_opd_methods/2026_microsoftresearch_self_distilled_rlvr_2604.03128.pdf`

### Q: `LLM_OPD失败模式_机制归因_算法改进映射.md` 中关于 `2604.03128` 的描述是否有误？

审核结论：原描述抓到了论文现象，但措辞过宽，容易误解成 “privileged teacher signal 本身不可用”。更准确的说法是：

```text
不是所有 privileged teacher signal 都必然导致泄漏；
问题出在 OPSD 的 distribution matching 目标中，
privileged teacher evaluation 进入了梯度方向。
```

论文确实报告了：

- OPSD 会出现 privileged-information leakage
- 模型会在推理时显式提到不可见的 reference solution
- OPSD 早期提升后后期性能下降
- OPSD KL 短暂下降后 stagnate

但机制归因更精确地是：OPSD 试图让 student 匹配 teacher 的条件分布：

$$
P_T(\cdot \mid x,r,y_{<t})
$$

其中 `r` 是 privileged information / reference trace，而 student 推理时只能表达：

$$
P_S(\cdot \mid x,y_{<t})
$$

论文把 OPSD loss 分解为：

$$
L_{\mathrm{OPSD}}
=
L^*
+
I(Y_t;R\mid X,Y_{<t})
$$

后面这个 mutual-information gap 是不可约的，因为 teacher 的 token prediction 依赖 `R`，但 student 无法访问 `R`。

更关键的是 per-sample gradient。虽然 population-level expected gradient 可能是 benign 的，但单样本梯度里有 `r`-specific deviation；只要 teacher 的 privileged evaluation 进入 gradient direction，就会形成 leakage channel。论文还用 full OPSD、teacher top-1、student-sampled 等变体说明：不同形式只是 leakage bandwidth 不同，方向依赖 `r` 这一点仍存在。

RLSD 的修复不是丢弃 privileged teacher，而是改变使用方式：

```text
OPSD:
  privileged teacher distribution 决定更新方向

RLSD:
  verifier / RLVR reward 决定更新方向
  privileged teacher evidence ratio 只调节更新幅度
```

RLSD 使用：

$$
\frac{P_T(y_t)}{P_S(y_t)}
$$

作为 stop-gradient scalar multiplier / evidence ratio，只调节 token-level credit magnitude；trajectory 是 reinforce 还是 penalize，仍由 verifier outcome / RLVR advantage 决定。

已据此修正：

- `docs/LLM_OPD失败模式_机制归因_算法改进映射.md`
- `docs/快速总结.md`
- `docs/论文类型总结.md`

修正后的口径：`2604.03128` 的 failure mode 是 **OPSD distribution matching under information asymmetry**，不是 PI 信号本身；RLSD 的贡献是 **direction / magnitude decoupling**，让 PI 只影响幅度，不控制梯度方向。

## 2604.03128 - Self-Distilled RLVR

本地文件：`../papers/llm_opd_methods/2026_microsoftresearch_self_distilled_rlvr_2604.03128.pdf`

### Q: “OPSD 的分布匹配目标在信息不对称下是 ill-posed：teacher 看到 privileged information / reference trace，student 推理时看不到；若直接匹配 `P_T(. given x,r,y_<t)`，会出现 privileged-information leakage、KL stagnation 和后期性能下降”是什么意思？为什么是在 teacher entropy 高时压缩多样性？

这句话前半部分对应 `2604.03128 Self-Distilled RLVR` 的核心论点；但后半句“teacher entropy 高时压缩多样性”不是这篇论文的机制，而是 `2603.07079 Entropy-Aware OPD` 的机制。两者不要混在一起。

`2604.03128` 讨论的是 OPSD 中 **信息不对称下的分布匹配失败**。

OPSD 里同一个模型扮演两种角色：

- privileged teacher：看到题目 `x` 和 privileged information / reference trace `r`
- student：推理时只看到题目 `x`

在 token `t` 的位置，teacher 的分布是：

$$
P_T(\cdot \mid x,r,y_{<t})
$$

student 的分布是：

$$
P_S(\cdot \mid x,y_{<t})
$$

普通 OPSD 试图直接做 distribution matching，让 student 匹配 teacher：

$$
P_S(\cdot \mid x,y_{<t})
\rightarrow
P_T(\cdot \mid x,r,y_{<t})
$$

问题是 `r` 在训练时给 teacher 看，但推理时 student 看不到。因此 teacher 的 token preference 可能依赖 `r` 中的信息，而 student 无法合法访问这些信息。

论文把 OPSD loss 写成：

$$
L_{\mathrm{OPSD}}
=
\mathbb{E}_{x}
\mathbb{E}_{r\sim P(r\mid x)}
\left[
D_{\mathrm{KL}}
\left(
P_T(\cdot \mid x,r)
\|
P_S(\cdot \mid x)
\right)
\right]
$$

理想上，student 不应该匹配每个具体 `r` 对应的 teacher posterior，而应该匹配把 `r` marginalize 掉之后、student 可表达的边际目标：

$$
\bar P_T(\cdot \mid x)
=
\mathbb{E}_{r\sim P(r\mid x)}
\left[
P_T(\cdot \mid x,r)
\right]
$$

论文证明 OPSD objective 可以分解为：

$$
L_{\mathrm{OPSD}}
=
L^*
+
I(Y_t;R\mid X,Y_{<t})
$$

其中：

- `L*` 是 student 能合法匹配的理想边际目标
- `I(Y_t;R | X,Y_<t)` 是当前 token 和 privileged information 之间的条件互信息 gap

只要 teacher 的 token prediction 依赖 `r`，这个互信息项就是正的：

$$
I(Y_t;R\mid X,Y_{<t})>0
$$

这就是论文说的 **ill-posed**：OPSD 要求 student 匹配一个依赖 `r` 的条件分布，但 student 的输入里没有 `r`。

这会带来三个现象。

**1. Privileged-information leakage**

因为训练梯度反复把 `r` 相关的偏好推给 student，student 会试图在参数里学习从 `x` 到 `r` 的隐式映射。论文展示了 OPSD-trained model 在推理时会显式提到不可见的 “reference solution”，这就是 leakage。

**2. KL stagnation**

因为存在不可约互信息 gap，student 无法真正把：

$$
P_S(\cdot \mid x)
$$

收敛到每个具体：

$$
P_T(\cdot \mid x,r)
$$

所以 OPSD 的 teacher-student KL 会先下降一小段，然后 stagnate。论文用这个解释 Figure 3 中 OPSD KL divergence 的停滞。

**3. 后期性能下降 / long-term instability**

论文进一步分析 per-sample gradient。虽然 population-level expected gradient 可能等价于理想边际目标，但单样本梯度里有 `r`-specific deviation。只要 teacher 的 privileged evaluation 进入 gradient direction，参数更新方向就被 `r` 污染。

这会形成反馈回路：

```text
teacher 依赖 r 的 token preference 进入梯度方向
student 参数开始编码 r 的痕迹
shared-parameter / online teacher 又被更新后的参数影响
模型对 privileged information 更敏感
leakage 和不稳定继续放大
```

所以这篇论文的关键不是 “teacher entropy 高”，而是：

```text
teacher 看到了 student 推理时看不到的信息 r
OPSD 又强行让 student 匹配这个 r-conditioned distribution
于是 r 进入了梯度方向
```

RLSD 的修复是 **方向 / 幅度解耦**。

它不再让 privileged teacher 决定 token 更新方向，而是让 verifier / RLVR outcome 决定方向：

```text
正确轨迹 -> reinforce
错误轨迹 -> penalize
```

privileged teacher 只提供 token-level evidence ratio，调节幅度：

$$
w_t
=
\left(
\frac{
P_T(y_t \mid x,r,y_{<t})
}{
P_S(y_t \mid x,y_{<t})
}
\right)^{\mathrm{sign}(A)}
$$

其中 `A` 来自 RLVR / verifier advantage。这样：

- `sign(A)` 决定 token 是被强化还是惩罚
- teacher evidence ratio 只决定 credit 有多大
- privileged information 不再决定梯度方向

因此 RLSD 仍然使用 privileged teacher 的信息，但把它从 “generative target” 改成 “stop-gradient scalar multiplier”。

至于“为什么是在 teacher entropy 高时压缩多样性”，这是 `2603.07079` 的问题，不是 `2604.03128` 的问题。

`2603.07079` 说的是：当 teacher distribution 高熵时，teacher 对多个 token 都给不低概率，代表多个合理 continuation；reverse KL 的 mode-seeking 性质只沿 student 采到的 token 更新，不会覆盖 student 没访问到的 teacher modes，因此会把 teacher 的多峰分布压成少数 dominant modes。

也就是：

```text
2603.07079:
  高 teacher entropy + reverse KL mode-seeking
  -> diversity compression

2604.03128:
  privileged teacher conditioning + OPSD distribution matching
  -> information leakage / KL stagnation / late-stage degradation
```

`2604.03128` 中确实也讨论 entropy dynamics，但重点是 GRPO 的 uniform sequence-level reward 容易 rapid entropy collapse，而 RLSD 通过 token-level credit reweighting 保持更高 entropy；这不是 “teacher entropy 高导致压缩多样性” 的机制。

一句话总结：`2604.03128` 的 failure mode 是 **信息不对称下把 `P_T(.|x,r)` 当作 student 可匹配目标**，导致 privileged information 进入梯度方向；“teacher entropy 高时 reverse KL 压缩多样性”属于 `2603.07079`，是另一种 KL 方向与 teacher uncertainty 不匹配的问题。

## 2601.18734 vs 2604.03128 - OPSD 到 RLSD 的关系

本地文件：

- `../papers/llm_opd_methods/2026_ucla_self_distilled_reasoner_on_policy_self_distillation_for_larg_2601.18734.pdf`
- `../papers/llm_opd_methods/2026_microsoftresearch_self_distilled_rlvr_2604.03128.pdf`

### Q: `2604.03128 Self-Distilled RLVR` 和 `2601.18734 Self-Distilled Reasoner` 是递进关系吗？

可以说是 **研究脉络上的递进关系**，但不是简单的“后一篇把前一篇算法直接加强一下”。

`2601.18734 Self-Distilled Reasoner` 提出的是 OPSD 这个正向范式：

```text
同一个模型分成 student / privileged teacher 两种上下文
student 只看题目 x
teacher 额外看到 reference solution / verified reasoning trace y*
student 生成自己的 rollout
teacher 在 student rollout 上给 dense token-level supervision
```

它的核心目标是解决普通 OPD 依赖外部大 teacher 的问题。形式上：

$$
p_S(\cdot \mid x,y_{<t})
$$

去匹配：

$$
p_T(\cdot \mid x,y^*,y_{<t})
$$

论文主实验采用 full-vocabulary logit distillation，也讨论 sampled-token reward 视角。它强调 OPSD 的优点：

- 不需要额外外部 teacher
- 利用 reasoning dataset 里的 reference solution
- 在 student 自己 rollout 上给 dense per-token signal
- token efficiency 高于 GRPO / RLVR

但 `2601.18734` 主要是在展示 OPSD 有效，并没有深入分析信息不对称会导致什么长期失败。它的 limitation 里反而指出未来可以引入 correctness verification，因为当前框架没有显式利用 generated answer 的 correctness。

`2604.03128 Self-Distilled RLVR` 则把这类 OPSD 当成研究对象，问的是：

```text
为什么普通 OPD 可以工作，而 OPSD 会失败？
```

它承认 OPSD 有 token efficiency，但指出在 privileged teacher 场景下，直接 distribution matching 会出现：

- privileged-information leakage
- teacher-student KL stagnation
- early peak 后 late-stage performance degradation
- online teacher 下长期训练不稳定

它的关键理论诊断是：OPSD 让 student 匹配依赖 privileged information `r` 的 teacher distribution：

$$
P_T(\cdot \mid x,r,y_{<t})
$$

但 student 推理时只能表达：

$$
P_S(\cdot \mid x,y_{<t})
$$

因此 OPSD objective 是 ill-posed。论文把它分解为：

$$
L_{\mathrm{OPSD}}
=
L^*
+
I(Y_t;R\mid X,Y_{<t})
$$

后面的 mutual-information gap 是 student 无法消掉的，因为 `R` 不在 student 输入里。更进一步，per-sample gradient 中有 `r`-specific deviation，teacher 的 privileged evaluation 会进入 gradient direction，从而引发 leakage。

所以二者的递进关系是：

```text
2601.18734:
  提出/验证 OPSD：让 privileged teacher 提供 dense token supervision

2604.03128:
  诊断 OPSD：当 teacher privilege 是 student 推理时不可见的信息时，
  distribution matching 本身会把隐藏信息注入梯度方向

  修复：RLSD 不再让 teacher 决定方向，
  而是让 RLVR / verifier outcome 决定方向，
  teacher evidence ratio 只调节 token-level 更新幅度
```

`2604.03128` 的 RLSD 可以看成对 `2601.18734` OPSD 的一个关键修正：

```text
OPSD:
  teacher = generative target
  student 模仿 P_T(. | x,r)

RLSD:
  teacher = evidence / credit evaluator
  verifier reward 决定 reinforce 还是 penalize
  P_T / P_S 只决定 credit magnitude
```

但它们不是完全同一个实验设置：

- `2601.18734` 主要是文本数学 reasoning，重点在 reference solution / verified trace 带来的 dense supervision。
- `2604.03128` 主要围绕 RLVR 和 multimodal reasoning，重点在 verifier outcome anchoring 与 leakage-free credit assignment。
- `2601.18734` 主要报告短程训练收益；`2604.03128` 更强调长期训练中的 leakage、KL stagnation 和后期退化。

因此，最准确的说法是：

```text
2601.18734 是 OPSD 范式的提出和有效性展示；
2604.03128 是对 OPSD 在 privileged-information setting 下的失败机制分析，
并提出 RLSD 作为 direction/magnitude decoupled 的修正版。
```

一句话总结：是递进关系。`2601.18734` 说明“reference-conditioned self-teacher 可以给 student dense supervision”；`2604.03128` 进一步指出“如果直接把这个 privileged teacher 当分布匹配目标，会因信息不对称导致泄漏和不稳定”，于是把 teacher signal 从更新方向中拿出来，只用于调节更新幅度。

## 2604.03873 - SODA: Semi On-Policy Black-Box Distillation

本地文件：`../papers/llm_opd_methods/2026_academic_soda_semi_on_policy_black_box_distillation_for_large_languag_2604.03873.pdf`

### Q: “黑盒 KD 面临两难：off-policy sequence KD 难纠正 student 自身错误；fully on-policy / adversarial distillation 能纠错但训练不稳且成本高”是什么意思？

这篇论文研究的是 **black-box knowledge distillation**。Black-box 的意思是：teacher 可能是 GPT-5-Chat 这类闭源 frontier model，训练者只能拿到 teacher 生成的文本回答：

$$
y^T \sim p_T(\cdot \mid x)
$$

拿不到：

- teacher logits
- teacher hidden states
- teacher gradients
- token-level probability distribution

因此不能做标准 white-box OPD 那种：

$$
D_{\mathrm{KL}}
\left(
p_T(\cdot \mid x,y_{<t})
\|
q_\theta(\cdot \mid x,y_{<t})
\right)
$$

在黑盒场景里，最常见方法是 **sequence-level KD / SeqKD**：让 teacher 对每个 prompt `x` 生成一个好答案 `y^T`，然后用 SFT 训练 student：

$$
\max_\theta
\log q_\theta(y^T \mid x)
$$

这就是论文说的 off-policy sequence KD。它的优点是简单、稳定、可扩展；缺点是完全只看 teacher demonstrations，训练时没有使用 student 自己会犯什么错的信息。

为什么这会“难纠正 student 自身错误”？

因为 SeqKD 的训练数据只包含：

```text
x -> teacher answer
```

没有包含：

```text
x -> student wrong answer
teacher answer 为什么比 student wrong answer 好
```

所以 student 只是被动模仿 teacher，不知道自己当前分布里哪些典型输出模式是坏的。论文强调两个问题：

1. **缺少 negative examples**

   SFT 没有机制把 teacher 的好回答和 student 的特征性错误做对比。

2. **exposure bias / distribution mismatch**

   训练时 student 总是在 teacher prefixes 上学；推理时却要沿着自己生成的 prefix 继续写。如果 student 早期生成偏了，SeqKD 不会告诉它如何从自己的错误状态中恢复。

这就是：

```text
off-policy SeqKD:
  学 teacher 怎么说
  但不直接惩罚 student 自己常犯的错
```

另一端是 **fully on-policy / adversarial distillation**，论文主要对比的是 GAD：Generative Adversarial Distillation。

GAD 的思路是把 student 当 generator，再训练一个 discriminator 来区分：

```text
teacher response
student response
```

student 每一步训练都从当前 policy 生成 responses，discriminator 给 sequence-level reward / score，student 再通过 adversarial RL 去改进。这样它确实使用了 student 当前分布，所以能针对 student 自己的错误给反馈。

但 fully on-policy GAD 的问题是：

- 需要额外 discriminator model
- generator / discriminator 要交替更新
- minimax adversarial training 本身不稳定
- reward / baseline / rollout 数量等超参难调
- 每个训练 step 要从当前 student 采样多个完整 responses
- sequence generation 本身很贵，叠加 discriminator 后内存和时间都很高

论文把这个说成黑盒 KD 的两难：

```text
SeqKD:
  稳定便宜
  但完全 off-policy，忽略 student 自己的错误

GAD:
  能 on-policy 地看到 student 当前错误
  但 adversarial training 不稳定，而且训练成本很高
```

SODA 的核心想法是：不必每一步都做 fully on-policy。对于 frontier teacher 和小 student，base student 的 zero-shot responses 通常已经明显弱于 teacher responses。因此可以一次性生成 student 的初始回答，把它当成固定 negative：

$$
y^0 \sim q_0(\cdot \mid x)
$$

teacher response 当 positive：

$$
y^T \sim p_T(\cdot \mid x)
$$

构造 preference pair：

$$
(x, y^T \succ y^0)
$$

然后先用 teacher response 做一个短 warmup：

$$
q_0 \rightarrow q_w
$$

再从 `q_w` 出发，用 DPO / preference optimization 训练：

```text
preferred: teacher response
rejected: base student response
```

这就是 SODA 的 semi-on-policy：

- **on-policy 的部分**：negative 来自 student 自己的分布 `q_0`，所以它包含 student 特征性错误。
- **off-policy / static 的部分**：这些 student negatives 只生成一次，训练中不随当前 policy 动态刷新。

论文用一句话概括这个位置：

```text
SeqKD ignores the student distribution entirely.
GAD continuously tracks the current student distribution.
SODA uses a one-time static snapshot of the student prior.
```

SODA 为什么能纠正 student 自身错误？

因为它的 preference pair 不是泛泛的 “teacher 好”，而是：

```text
teacher response 比这个 student 自己会生成的 response 好
```

这给了一个 student-specific contrastive signal。相比随机 bad response 或 generic negative，`q_0` 生成的 negative 更有针对性，因为它来自这个 student 真实会访问的错误区域。

可以理解为：

```text
SeqKD:
  学 what teacher says

SODA:
  学 what teacher says
  同时学 what this student should stop saying
```

论文称其为 dual learning signal：

- teacher-positive term：teacher imitation
- student-negative term：penalize persistent inferior behaviors

为什么 SODA 比 fully on-policy 更稳定、更便宜？

因为它去掉了：

- per-step dynamic rollout
- discriminator
- adversarial minimax
- RL reward baseline
- 多轮 generator/discriminator balancing

训练数据在开始时就构造好：

$$
D_{\mathrm{pref}}
=
\{(x_i, y_i^T, y_i^0)\}
$$

后续就是普通 preference distillation。论文报告 SODA 在质量上匹配或超过 GAD，同时训练约 `10x` 更快、显存更低。

一句话总结：这篇论文的意思是，黑盒 KD 中 SeqKD 太“离线”，只模仿 teacher，不能针对 student 自己的错误；GAD 太“在线”，能纠错但要持续 rollout 和 adversarial training，成本高且不稳。SODA 折中：一次性采样 base student 的错误输出作为固定 negative，用 teacher output 作为 positive 做 preference distillation，从而获得 student-aware 的纠错信号，同时避免 fully on-policy adversarial 训练的开销和不稳定。

## 2604.04461 - DP-OPD: Differentially Private On-Policy Distillation for Language Models

本地文件：`../papers/llm_opd_methods/2026_academic_dp_opd_differentially_private_on_policy_distillation_for_lan_2604.04461.pdf`

### Q: “私有语料适配中，DP-SGD 会带来 utility loss，长 autoregressive rollout 中噪声会放大 exposure bias；已有私有蒸馏要么训练 DP teacher 成本高，要么依赖 DP synthetic text pipeline”是什么意思？

这篇论文讨论的是 **在私有语料上训练 / 压缩语言模型**。目标是既要保护训练数据隐私，又要得到可用的 student model。

这里的隐私保护主要指 **differential privacy (DP)**。在深度学习中常用 **DP-SGD** 实现：对每个样本的梯度做 clipping，然后加高斯噪声：

$$
g_i \leftarrow \mathrm{clip}(g_i, C)
$$

$$
\bar g
=
\frac{1}{B}
\left(
\sum_i g_i
+
\mathcal{N}(0,\sigma^2 C^2 I)
\right)
$$

这样最终发布的 student 参数满足 record-level privacy guarantee，例如论文实验里的：

$$
\epsilon=2.0,\qquad \delta=1/N
$$

但 DP-SGD 的问题是：噪声会损伤优化质量，所以通常会有 **utility loss**。在语言模型里，这个问题更明显，因为 autoregressive generation 是一步错、后面都可能跟着偏。

普通 teacher-forcing / off-policy KD 的训练状态是：

```text
训练时：
  student 在真实文本 prefix / teacher prefix 上学

推理时：
  student 在自己生成的 prefix 上继续生成
```

如果 student 在推理时前面生成错了，后面进入的 prefix state 训练时没见过，错误会继续累积。这就是 exposure bias / compounding errors。

在 DP 场景下，DP-SGD 噪声会让 student 的局部预测更容易偏掉；一旦偏掉，长 rollout 中偏差会被 autoregressive 机制放大：

```text
DP noise -> 局部 token 预测更差
局部错误 -> 后续 prefix 分布偏移
分布偏移 -> 后续 token 更难预测
长序列中错误累积
```

所以那句话里的 “长 autoregressive rollout 中噪声会放大 exposure bias” 指的是：DP-SGD 的优化噪声让 student 更容易在自己生成轨迹上进入错误状态，而 off-policy / teacher-forced 训练并没有在这些状态上给它 teacher feedback。

已有私有蒸馏路线主要有两类。

第一类是 **dual-DP teacher-student distillation**。

做法是：

```text
先用 DP-SGD 训练 / 微调 teacher
再用 DP-SGD 训练 student
```

问题是：

- 大 teacher 做 DP-SGD 很贵
- teacher 和 student 都用 DP，会叠加计算负担
- 严格隐私预算下 privacy-utility tradeoff 更差

第二类是 **DP synthetic text pipeline**，论文主要对比 DISTILDP。

做法是：

```text
1. 用 DP-SGD 训练一个 DP teacher
2. 用 DP teacher 生成 synthetic text
3. 因为 synthetic text 是 DP teacher 的 post-processing，所以可认为带 DP
4. student 在 synthetic corpus 上非私有训练 / 蒸馏
```

这个方法的好处是 student 不必直接做 DP-SGD；但代价是：

- 仍然要 DP 训练一个大 teacher
- 要额外生成大规模 synthetic text
- decoding strategy、synthetic set size、prompt 设计都会影响结果
- synthetic data 和真实私有语料、student 推理分布之间又会产生新的 distribution shift
- pipeline 更复杂

DP-OPD 的核心改法是：**只对 student updates 做 DP-SGD，teacher 冻结且不发布；teacher 只在训练内部给 student rollout 提供 dense token targets。**

流程是：

```text
private corpus D
student 在 prompt 上生成自己的 continuation
frozen teacher 在 student-generated states 上给 logits / token targets
student 用 distillation loss 更新
student 更新通过 DP-SGD 加噪和裁剪
最终只发布 student 参数
```

形式上，student rollout 是：

$$
y \sim S_\theta(\cdot \mid x)
$$

teacher 在 student 真实访问的 prefix 上给分布：

$$
T(\cdot \mid x,y_{<t})
$$

student 用 generalized KD loss 学：

$$
L_{\mathrm{GKD}}
\left(
S_\theta(\cdot \mid x,y_{<t}),
T(\cdot \mid x,y_{<t})
\right)
$$

然后这一步 student update 用 DP-SGD：

$$
\theta \leftarrow \mathrm{DP\text{-}SGD}(\nabla_\theta L_{\mathrm{GKD}})
$$

这就是 “privacy placement” 的关键：

- teacher 是 frozen public / independently trained model
- teacher inference 是训练内部计算
- teacher 不用 DP-SGD
- teacher 不发布
- 隐私预算只由 student 对 private dataset 的 DP-SGD 更新消耗

论文明确说：teacher inference does not consume additional privacy budget，因为最终发布的只有 student 参数，DP accounting 完全来自 student updates。

为什么 on-policy 对 DP 特别有用？

因为 DP 噪声会让 student 的状态分布更容易偏离 teacher-forced data。DP-OPD 直接在 student 自己访问的 states 上问 teacher：

$$
(x,y_{<t}) \sim \text{student rollout states}
$$

而不是只在固定真实文本 / teacher-forced trajectory 上蒸馏。这样 teacher correction 发生在 student 真正会出错、会访问的地方，所以能缓解 DP 噪声放大的 exposure bias。

可以压缩成：

```text
off-policy DP KD:
  在固定文本 prefix 上学 teacher
  DP noise 导致 student 推理时偏离后，缺少纠正

DP-OPD:
  student 自己 rollout
  teacher 在 student-visited states 上给 dense targets
  DP-SGD 只保护 student update
```

论文实验在严格隐私预算：

$$
\epsilon=2.0
$$

下比较：

- student-only DP-SGD
- dual-DP / off-policy private distillation
- DP synthetic text distillation
- DP-OPD

结果是 DP-OPD 在 Yelp 和 BigPatent 上 perplexity 更好，并且 pipeline 更简单。

一句话总结：这篇论文的意思是，私有 LM 训练里 DP-SGD 的噪声会降低生成质量，并在长自回归生成中放大 exposure bias；传统私有蒸馏要么把 DP 也加到大 teacher 上，成本和 utility 都不理想，要么先造 DP synthetic text，pipeline 复杂且有分布偏移。DP-OPD 把隐私保护只放在 student DP-SGD 上，同时让 frozen teacher 在 student 自己 rollout 的状态上给 dense targets，从而兼顾隐私、teacher guidance 和 on-policy 纠错。
