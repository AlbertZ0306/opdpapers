## 1. 口径与约束

现有 OPD 实验的现实条件（决定哪些 idea 能立刻起步）：

- 主流 setup：Qwen3 / OLMo 系，1.5B–8B，math reasoning（AIME/MATH/LiveCodeBench），GRPO-style RLVR + token-level KL OPD。
- 多数方法可在单机多卡上跑通；teacher 通常是 white-box，可拿 logits。
- 共同短板：跨家族、长 horizon、agentic、多模态、subjective 任务都验证不足。

标注约定：**[即试]** 现有 setup 下可直接起步；**[需搭基建]** 需要先做 benchmark / counterfactual pipeline 等前置工作。

## 2. 现状收敛出的 5 个核心空白

现状文档反复指向的"最缺"，是所有 idea 的靶子：

1. **OPD 适用条件预测器**（go/no-go detector）—— 训练前判断这个 teacher-student-data 组合是否值得做 OPD。
2. **teacher signal reliability estimator** —— entropy / PPL / top-k overlap / fidelity / KL agreement 都只是 proxy，缺统一可靠性度量。
3. **token-level 因果 credit assignment** —— correctness 是 trajectory-level，teacher KL 是 token-level，二者不必因果一致。
4. **black-box dense teacher signal** —— 从 response / rubric / verbal uncertainty 恢复近似 logits/uncertainty。
5. **long-horizon agentic OPD 理论与 benchmark** —— prefix drift / SFD / KL agreement trap 缺统一评测。

> ⚠️ 核查教训：这 5 个空白被"survey 点名为 open"恰恰意味着很多人在抢。其中 #1（go/no-go 诊断）已被 Unmasking OPD `2605.10889` 用梯度-cosine 占据，#2/#6 的多数子问题也已有论文。**P 系列特意避开"补空白"的正面打法，改走三招中的去魅 / 前沿 / 混淆分析。**

## 3. P 系列：冲顶会主线

### ⭐ P5：OPD 的收益是知识迁移还是方差缩减？（已成文）

**一句话论点**：*Is On-Policy Distillation Knowledge Transfer or Variance Reduction?* —— 标准推理 OPD 的大部分增益来自对稀疏 RL credit 的**方差缩减**，而非 teacher 分布带来的新知识；一个**零外部知识**的 teacher-free 方差缩减 RLVR 能复现其中 $\rho$ 比例的增益。

**为什么躲过饱和**：核查确认 control-variate 视角与"方差缩减 vs 知识迁移"归因无人做过；卖点是**经验去魅**而非框架，绕开"统一已被预占"的死穴；且 **$\rho$ 高/低两种结果都能发**。复用 N1 的 control-variate 机器、方向力度解耦的 factorial design、反事实 pipeline。

**完整设计**（teacher-free 方差缩减基线 B1/B2/B3、复现率 $\rho$、覆盖集解锁探针、regime 相图、kill test）见 → **[OPD收益归因_方差缩减还是知识迁移.md](/data1/zhaoenyue/opdpapers/docs/04想法/OPD收益归因_方差缩减还是知识迁移.md)**。

---

### P1：OPD 的 token-level credit 是被混淆的——一个因果审计

**一句话论点**：*Token-Level Credit in OPD Is Confounded* —— 现有所有 token 重要性信号（teacher-student divergence、entropy、teachability、PMI、position）都是**相关性 proxy**；用 interventional rollout 测**真实因果效应**，证明这些 proxy 与因果收益相关性很弱，真正有因果作用的是一小撮被 proxy 漏掉的 token，一个廉价的因果感知 credit 用少量 token 即可复现大部分增益。

**为什么躲过饱和**（招 i：攻击共享假设）：整个 token-selection 子领域默认"proxy ≈ 因果重要性"。核查**没碰**因果这条线；repo 把"token 因果 credit"明列为**仍待解决**，现存全是相关性 proxy（TIP `2604.14084`、Token Teachability `2605.26844`、PMI `2605.11609`、Input-Specific Credit `2605.11613`）——**没人拿到因果 ground truth 去检验这些 proxy 准不准**。不吃 RLSD/Unmasking 的预占。

**核心做法**：
1. 在 student prefix $s_{i,t}$ 上，用 interventional rollout 估计每个 token 的**真实因果 advantage** $\hat A^{*}_{i,t}=\hat Q-\hat V$（强制该 token vs 不强制，各续写 $K$ 次，verifier outcome 之差）。
2. **诊断（论文核心）**：用各 proxy 预测 $\hat A^{*}$ 的符号/幅度，报 AUC / 秩相关——预期得到"proxy 与因果效应弱相关"的**反直觉负结果**。
3. **修复（兜底）**：用因果标签蒸馏一个 cheap causal-aware credit predictor，做 token gating，对比 TIP / Token Teachability 的 proxy gating，证明少量因果 token 复现大部分增益。

**与 P5 的关系**：互补。P5 问"teacher 信号整体是否=方差缩减"；P1 问"token 级选择信号是否=因果"。两者共用 interventional rollout pipeline（N1 §6 / Idea 1）。

**可行性**：中（反事实续写成本，离线子集做一次）。**风险**：审稿人可能要"修复打过 SOTA"，故必须带兜底方法；$\hat A^{*}$ 噪声 → 主结论只依赖秩相关而非精确值。**前身**：Idea 1（Counterfactual Teachability）+ N3（因果升级）的合并升级版。

---

### P2：面向工具调用 Agent 的 on-policy distillation

**一句话论点**：*On-Policy Distillation for Tool-Using Agents: Credit Beyond Tokens* —— 现有 OPD 都假设一条**文本轨迹**；agent 的轨迹是 (action, tool-result, env-state)，teacher 监督与 credit 必须落在**决策点**而非 token，且 exposure bias / prefix drift **跨 turn 复利放大**。给出第一个原理性的 agent-OPD 方法 + 评测。

**为什么躲过饱和**（招 ii：无人前沿）：核查与 repo **一致确认**所有论文把 agentic OPD 推给 future work；2026 = agent 年，timely 拉满。三个 P 里**天花板最高**。

**核心做法**：
- 把监督单元从 token 上移到 **(决策, 工具结果) 段**：teacher 在 student 真实交互轨迹上对**动作选择**给 dense 信号，工具返回当作不可控的环境观测（不蒸馏）。
- credit 跨 turn 分配：把 N1 的 advantage 分解扩到"哪一步工具调用 / 哪一段推理"导致最终成败。
- 处理跨 turn 复利的 exposure bias：长交互里坏的早期决策会污染所有后续 teacher 信号（prefix drift 的 agent 版）。

**可行性**：中偏重（需 agent 环境，如工具使用 benchmark）。**风险**：state/credit 单元的定义是真难点；执行重。建议**聚焦单一工具环境**先做出存在性证明，而非通用框架。

---

### P4：OPD 的"内化"到底往权重里写了什么——机制审计

**一句话论点**：*What Does On-Policy Distillation Write Into the Weights?* —— `2606.11627`（When Context Returns）发现一个反直觉现象：student 内化 privileged context 后，**推理时重新给回该 context 反而变差**。这说明"内化"不是简单学会"有无 context 都稳"。我们做一个机制/表征审计，回答内化是**学到可迁移能力**还是**记住了一个 shortcut**。

**为什么躲过饱和**（机制 interp）：现象已发表（`2606.11627`）但**机制无人解**；ICLR 偏好这类"解释一个反直觉现象"的 interp 论文。

**核心做法**：表征探针 + 因果干预，定位 OPD 在哪些层/方向写入了"context-conditioned 行为"，检验它与"内化为参数知识"是否冲突（context-return 退化的机制根因）。

**可行性**：中（interp 方法不确定性高）。**风险**：interp 论文成败取决于能否给出**干净可验证**的机制，否则沦为定性叙事。三个 P 外候选里风险最高。

---

### P3：正确轨迹上 teacher 到底压制、压缩还是纠错（机制澄清）

**一句话论点**：*When Does the Teacher Compress, Suppress, or Correct?* —— 三篇论文在"正确 rollout 上 teacher 的作用"上**互相矛盾**：RLRT（`2605.10781`）说 teacher **压制** student 成功路径、应反向用；OPSD Compresses（`2605.06188`）说是**安全压缩**；Unmasking（`2605.10889`）说 teacher 在正确轨迹上**变噪声**。三者不能同真。我们用控制实验拆解、给统一机制与干预。

**为什么躲过饱和**：有真实的论文间矛盾待解。但**与现有 correct/wrong 工作重叠较多**，是 P 系列里新颖性最低的一条。

**与 P5 的关系**：互补且更细——P5 问"VR vs KT"的全局归因，P3 钻"正确轨迹内部"的三态。可作为 P5 的一个深化章节。**可行性**：中。

## 4. 基础候选库（8 个 building-block ideas）

这 8 个是最初按"可行性"梳理的候选。**它们不是冲顶会的主线**（多为 study / benchmark / 工程化路由，单独投难进主轨），但价值有二：(a) 部分已被 P 系列吸收/升级；(b) 其余是大论文的组成部件或 workshop / D&B track 的独立工作。每条标注其**归属**。

### Idea 1 [需搭基建]：Counterfactual Teachability
反事实 rollout 给 teacher token 打因果标签，检验哪些 cheap proxy 真能代表因果收益。对应空白 #2/#3。**归属：已升级为 P1**（P1 = Idea 1 + N3 因果框架）。

### Idea 2 [即试]：Correction/Compression/Reinforcement 三态在线检测器
对每个 OPD update 用短续写 outcome 判别是纠错/压缩/强化错误，据此 gating。差异于 OPSD Compresses 的静态分组。成本低、可复用 SCOPE/OPSD 代码。**归属：可作为 P3 的判别器、或 Idea 8 调度器的信号源；单独是 workshop 级。**

### Idea 3 [即试]：Estimator 等价性受控研究
固定 setup 全因子对比 full-vocab / top-k / sampled-token × FKL/RKL/JSD × explicit-KL/score-function/reweighting，产出偏差-方差-成本 Pareto。对应空白 #6。**归属：被 N1/P5 吸收为实证臂；单独是干净的 study（TMLR/workshop）。**

### Idea 4 [需搭基建]：Long-Horizon OPD 诊断 Benchmark
统一 prefix drift / SFD / KL agreement trap / recoverability / 后缀关键信号的探针，让 Prune-OPD/SFD/KAT/TRD 第一次可比。对应空白 #5。**归属：独立的 NeurIPS D&B track 候选；公共基建复用价值最高。**

### Idea 5 [需搭基建]：Black-Box 多样本语义分布恢复
在分支点对黑盒 teacher 采样 N 条续写构造经验分布，用 white-box oracle 量化逼近误差，回答"语义信号何时等价于 dense KL"。对应空白 #4。**归属：独立专题线（中等顶会潜力，niche）。**

### Idea 6 [即试→需基建]：统一 Teacher-Signal Reliability Router
把多个 reliability 信号当特征学一个 router，对每 token/prefix 输出 `{RL方向, OPD方向, OPD力度, 反向teacher, MLE, 截断, skip}`，监督信号用 Idea 1 因果标签。对应空白 #1/#2。**归属：方向力度解耦的上层泛化；无理论支撑会被当工程，建议作为 P1/P5 之上的方法章节。**

### Idea 7 [需搭基建]：Privileged Context Governance 统一研究
PI 类型 × 治理手段（exposure ratio / residual / evidence mask / removability）交叉评测，含 context-return robustness 与 leakage bandwidth。对应空白 #5(PI)。**归属：taxonomy 为主，缺 method 难顶会；可并入 P4（内化机制）。**

### Idea 8 [需搭基建]：OPD/RLVR 在线调度器
把"何时探索/压缩/纠错"建成 state-dependent 闭环控制，信号复用 Idea 2 三态检测。**归属：工程重、依赖 Idea 2；可作为 P5 结论的应用延伸。**

### 基础候选库的依赖关系


## 5. 演化记录（三次排序的推理链）

保留三轮迭代，便于追溯"为什么排序变了"。

### 第一轮（2026-06-22）：按"可行性"排序 8 个 idea

目标是"适合现在尝试"。结论：先做 **Idea 3**（零基建、必出对照表），高回报押 **Idea 1 → Idea 6**（因果标签 → reliability router）。详细优先级矩阵见基础候选库（§4）。

### 第二轮：改为"冲顶会"，提出 N1/N2/N3

重评后发现 8 个 idea **没有一个是拿来即冲顶会的形态**（多为 study/benchmark/工程路由），遂提出三个 flagship 级候选：

- **N1 方差-偏差分解**（最推荐）：把 RLVR 与 OPD 写成同一 PG estimator，证明 teacher 起 **control variate** 作用，OPD 效果 = 方差缩减项（不动 fixed point）+ 偏差/方向项（移向 teacher）；判据 OPD 有用 ⟺ 方差缩减 > 偏差²，取决于可测 `Cov(teacher, advantage)`。声称一次性解释 OPSD Compresses / RLSD / vOPD / capability ceiling。→ [OPD方差偏差分解理论.md](/data1/zhaoenyue/opdpapers/docs/04想法/OPD方差偏差分解理论.md)
- **N2 预测相图**：OPD/RLVR/hybrid 三区相界 + scaling-law 式 go/no-go 预测律。天花板高但**算力门槛硬**。
- **N3 Confounded Credit**：Idea 1 的严格因果升级（→ 已并入 **P1**）。

当时排序：N1 > N3 > N2 > Idea 1 > Idea 3 > 其余。**建议做 N1。**

### 第三轮（2026-06-23）：带引用 novelty 核查 → 重排为 P 系列

对 N1 做 deep-research 核查（5 角度、16 主源、25 条断言 3 票对抗验证），**关键发现**：

- N1 的**机制**（control-variate + 偏差-方差分解 + 协方差判据）3-0 **未被占据**；
- 但 N1 要统一的**四个现象全部已发表**：PG 统一（RLSD `2604.03128`、SRPO `2604.02288`、OPSD `2601.18734`、G-OPD `2602.12125`）、方向-力度（RLSD `2604.03128` 核心方法）、correct/wrong 非对称（Unmasking `2605.10889`、SCOPE `2604.10688` 等）、capability ceiling/cliff（G-OPD `2602.12125`）；
- "预训练 go/no-go 诊断"这一目标已被 **Unmasking OPD `2605.10889`** 用梯度-cosine 占据；
- 最接近的结构类比是 RLSD Prop 1 的 per-sample 梯度分解（但框成 MI leakage，非 control-variate）。

**结论**：N1 的"grand unification"是其最被预占的部分，单独投风险高 → 降级为 **P5 的理论附录**，用其唯一幸存的新资产（control-variate 视角）支撑 P5 的去魅型经验论断。并据"饱和三招"重想出 P 系列（见 §0、§3）。

## 6. 决策建议

1. **立刻做 P5 的 kill test**（半天）：在现成 OPD 日志上看 teacher-free dense baseline 能复现多少增益。
2. **结果好（$\rho$ 高、覆盖集不扩）** → 全力投 **P5**（形态 A，deflationary）。
3. **结果反向（$\rho$ 低、覆盖集扩）** → 切 P5 **形态 B**（"teacher 何时真做知识迁移"的正面结论），或转 **P1**（因果 credit 审计，与 P5 共用反事实 pipeline）。
4. **若想要更高天花板且有 GPU/工程预算** → 并行探索 **P2（Agent-OPD）**，它是唯一的"无人前沿"型，但执行最重。
5. **投稿前重跑一次 novelty 核查**——领域极快，结论时效性强。
