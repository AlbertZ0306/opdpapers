# Idea: 拆解 OPD 收益来自更新方向还是更新力度

更新时间：2026-06-11

## 背景

根据 [后续问答记录.md](/data2/zhaoenyue/opdpapers/docs/后续问答记录.md) 中关于“更新方向 / 更新力度”的讨论，可以把大模型后训练的梯度近似写成：

$$
\nabla_\theta L
\approx
\sum_i
c_i g_i
$$

其中：

- `g_i` 是更新方向，例如 sampled token 的 `\nabla_\theta \log \pi_\theta(a_t\mid s_t)`，或 full-vocab KL 的 distribution-level gradient；
- `c_i` 是更新力度，例如 outcome advantage、teacher-student log-ratio、token weight、confidence、clip 后系数；
- optimizer、learning rate、gradient clipping、batch averaging 再决定最终参数步长。

从这个角度看，RLVR 和 OPD 的区别不只是“用不用 teacher”，而是：

```text
RLVR / GRPO:
  方向 = student sampled token
  力度 = outcome reward / group-relative advantage

sampled-token OPD:
  方向 = student sampled token
  力度 = teacher-student log-ratio

full-vocab / top-k OPD:
  方向 = teacher-student distribution gap
  力度 = probability gap / KL weight / token mask

RLSD / RLRT:
  方向和力度被显式解耦
```

这自然引出一个研究问题：

```text
OPD 相比 RLVR 的收益，主要来自哪里？

1. teacher 改变了更新方向？
2. teacher 提供了更细粒度的更新力度 / credit assignment？
3. 二者都重要，但在不同 rollout 类型或 token 区域中作用不同？
```

## 核心假设

可以把 OPD 的收益拆成两个来源：

**假设 A：方向收益**

OPD 好，是因为 teacher 指出了 RLVR 不会更新到的方向。例如：

- full-vocab KL 能提高 student 没采到但 teacher 支持的 token；
- top-k FKL 能在 non-positive advantage 区域给出正确替代 token；
- teacher distribution matching 能修正 student 采样不到的 reasoning branch。

如果这个假设成立，那么只改变力度、不改变方向的 sampled-token OPD 或 RLSD-style magnitude modulation 收益有限；真正有用的是 full-vocab / top-k / teacher-target 方向。

**假设 B：力度收益**

OPD 好，主要是因为 teacher 给了更细的 token-level credit，而不是因为它改变了方向。例如 sampled-token OPD 和 RLSD 都仍沿着 student sampled token 更新，但用 teacher log-ratio / evidence ratio 替代稀疏 outcome advantage，能告诉模型：

```text
同一条正确或错误 rollout 中，哪些 token 更该被强化或削弱。
```

如果这个假设成立，那么在保持 RLVR 更新方向不变的情况下，只用 OPD 信号调节力度也能获得大部分收益。

**假设 C：条件依赖**

OPD 的方向收益和力度收益分别出现在不同区域：

```text
正确 rollout:
  RLVR 方向通常可靠
  teacher 更适合做力度校准，或甚至反向使用 teacher mismatch

错误 rollout:
  RLVR outcome 只知道整条错
  teacher 可能需要提供替代方向

高 student entropy token:
  teacher direction / top-k distribution 可能有帮助

低 entropy 但 teacher-student divergence 高的 token:
  teacher signal 更像纠错力度或反向压制信号
```

这个假设最接近 SCOPE、AOPD、RLSD、RLRT、TIP 等论文的共同趋势：不要全局统一使用 OPD，而要按 rollout correctness、advantage sign、teacher entropy、student entropy、teacher-student divergence 路由。

## 统一记号

student rollout：

$$
y_i\sim \pi_\theta(\cdot\mid x)
$$

token prefix / state：

$$
s_{i,t}=(x,y_{i,<t})
$$

student sampled token：

$$
a_{i,t}=y_{i,t}
$$

RLVR / GRPO 的 trajectory-level advantage：

$$
A_i^{\mathrm{RL}}
=
\frac{
R_i-\mathrm{mean}(R_1,\dots,R_G)
}{
\mathrm{std}(R_1,\dots,R_G)+\epsilon
}
$$

sampled-token OPD log-ratio：

$$
A_{i,t}^{\mathrm{OPD}}
=
\log \pi_T(a_{i,t}\mid s_{i,t})
-
\log \pi_\theta(a_{i,t}\mid s_{i,t})
$$

teacher distribution direction：

$$
g_{i,t}^{\mathrm{dist}}
=
\nabla_\theta
D_{\mathrm{KL}}
\left(
\pi_\theta(\cdot\mid s_{i,t})
\;\|\;
\pi_T(\cdot\mid s_{i,t})
\right)
$$

sampled-token likelihood direction：

$$
g_{i,t}^{\mathrm{sample}}
=
\nabla_\theta
\log \pi_\theta(a_{i,t}\mid s_{i,t})
$$

## 方向和力度的可组合矩阵

可以把训练 update 写成：

$$
\Delta\theta
\propto
\sum_{i,t}
c_{i,t}
g_{i,t}
$$

其中方向 `g` 和力度 `c` 分别从 RLVR 或 OPD 取。

### 1. Baseline 组合

| 名称 | 方向 `g` | 力度 `c` | 对应方法 | 检验意义 |
|---|---|---|---|---|
| RLVR / GRPO | sampled token direction | outcome advantage | GRPO / DAPO / Dr. GRPO | 稀疏 reward baseline |
| sampled-token OPD | sampled token direction | teacher-student log-ratio | G-OPD、REOPOLD、Lightning OPD | teacher 是否主要提供 token-level 力度 |
| full-vocab OPD | distribution KL direction | KL coefficient / probability gap | OPSD full-vocab、MAD-OPD | teacher 是否主要提供更新方向 |
| top-k OPD | subset distribution direction | subset probability gap / mask | OPCD、EOPD、AOPD、TIP | teacher direction 是否只需局部子集 |

### 2. 方向 / 力度交叉组合

| 实验组 | 方向来源 | 力度来源 | 形式 | 主要检验 |
|---|---|---|---|---|
| A0: GRPO | RLVR sampled token | RLVR outcome advantage | `A_i^RL * grad log pi(a_i,t)` | 纯 RLVR 基线 |
| A1: OPD magnitude only | RLVR sampled token | OPD log-ratio / teacher evidence | `f(A_i^RL, A_i,t^OPD) * grad log pi(a_i,t)` | OPD 是否主要靠力度校准 |
| A2: OPD direction only | teacher distribution direction | RLVR outcome advantage | `A_i^RL * grad KL(pi_theta, pi_T)` | OPD 是否主要靠方向 |
| A3: full OPD | teacher distribution or sampled-token OPD direction | OPD log-ratio / KL gap | 标准 OPD / top-k OPD | OPD 全量效果 |
| A4: hybrid routing | correct 用 RLVR direction，wrong 用 teacher direction | correctness / PPL / log-ratio | SCOPE/AOPD-style | 不同轨迹是否需要不同来源 |
| A5: reversed teacher on success | RLVR correct sampled token | reversed teacher-student ratio | RLRT-style | teacher mismatch 是否能成为探索力度 |
| A6: RL direction + teacher magnitude | RLVR direction | clipped teacher evidence ratio | RLSD-style | PI teacher 是否只应调力度 |

### 3. 具体 loss 设计

**A0: 纯 GRPO**

$$
J_{\mathrm{GRPO}}
=
\mathbb{E}
\sum_{i,t}
A_i^{\mathrm{RL}}
\log \pi_\theta(a_{i,t}\mid s_{i,t})
$$

方向：

$$
g_{i,t}=g_{i,t}^{\mathrm{sample}}
$$

力度：

$$
c_{i,t}=A_i^{\mathrm{RL}}
$$

**A1: RLVR 方向 + OPD 力度**

保持 sampled token direction 不变，只把 OPD log-ratio 当 token-level magnitude：

$$
J_{\mathrm{RLdir+OPDmag}}
=
\mathbb{E}
\sum_{i,t}
\hat A_{i,t}
\log \pi_\theta(a_{i,t}\mid s_{i,t})
$$

其中：

$$
\hat A_{i,t}
=
A_i^{\mathrm{RL}}
\cdot
\operatorname{clip}
\left(
\exp(\alpha A_{i,t}^{\mathrm{OPD}}),
1-\epsilon,
1+\epsilon
\right)
$$

或更简单：

$$
\hat A_{i,t}
=
A_i^{\mathrm{RL}}
+
\lambda A_{i,t}^{\mathrm{OPD}}
$$

这个组检验：

```text
如果 A1 接近或超过 full OPD，说明 OPD 主要收益来自 token-level 力度 / credit assignment。
```

注意：如果存在 privileged information，建议采用 RLSD 思路，让 RLVR 决定符号方向，teacher 只调 magnitude，避免 PI leakage。

**A2: OPD 方向 + RLVR 力度**

用 teacher distribution direction，但用 RLVR outcome advantage 控制更新是否发生和多大：

$$
L_{\mathrm{OPDdir+RLmag}}
=
\mathbb{E}
\sum_{i,t}
w(A_i^{\mathrm{RL}})
D_{\mathrm{KL}}
\left(
\pi_\theta(\cdot\mid s_{i,t})
\;\|\;
\pi_T(\cdot\mid s_{i,t})
\right)
$$

其中：

$$
w(A_i^{\mathrm{RL}})
=
\max(A_i^{\mathrm{RL}},0)
$$

或对错误轨迹取不同符号 / 不同 divergence：

$$
w(A_i^{\mathrm{RL}})
=
\left|A_i^{\mathrm{RL}}\right|
$$

这个组检验：

```text
如果 A2 明显优于 GRPO，而 A1 不明显，说明 teacher direction 本身很关键。
```

风险是：错误 rollout 的 prefix 可能 flawed，teacher distribution direction 也可能 noisy。SCOPE 的 flawed prefix trap 正是这个问题。

**A3: 标准 OPD / full OPD**

full-vocab reverse KL：

$$
L_{\mathrm{RKL}}
=
\mathbb{E}_{i,t}
D_{\mathrm{KL}}
\left(
\pi_\theta(\cdot\mid s_{i,t})
\;\|\;
\pi_T(\cdot\mid s_{i,t})
\right)
$$

或 sampled-token OPD：

$$
J_{\mathrm{sampled\ OPD}}
=
\mathbb{E}
\sum_{i,t}
A_{i,t}^{\mathrm{OPD}}
\log \pi_\theta(a_{i,t}\mid s_{i,t})
$$

这个组是 OPD baseline，用于和 A1 / A2 比较。

**A4: correctness-aware routing**

按 rollout 正误分流：

```text
correct rollout:
  不盲目模仿 teacher
  使用 RLVR / MLE / RLRT-style self-driven update

wrong rollout:
  使用 teacher direction 或 teacher magnitude 做纠错
```

可写成：

$$
J_{\mathrm{route}}
=
\mathbb{E}
\left[
\sum_{i\in\Omega_x^c,t}
c_{i,t}^{c}
g_{i,t}^{\mathrm{RL/self}}
+
\sum_{i\in\Omega_x^w,t}
c_{i,t}^{w}
g_{i,t}^{\mathrm{teacher}}
\right]
$$

其中：

- `Omega_x^c` 是正确轨迹集合；
- `Omega_x^w` 是错误轨迹集合；
- 正确轨迹可用 student PPL、RLRT ratio 放大低置信但成功的 token；
- 错误轨迹可用 teacher PPL、teacher entropy、teacher-student divergence 过滤 noisy teacher signal。

这个组检验：

```text
OPD 的收益是否只在错误轨迹 / 高不确定 token / teacher reliable prefix 上出现。
```

**A5: 反向 teacher signal on correct rollouts**

对正确 rollout：

$$
w_{i,t}^{\mathrm{RLRT}}
=
\left(
\frac{
\pi_\theta(a_{i,t}\mid s_{i,t})
}{
\pi_T(a_{i,t}\mid s_{i,t})
}
\right)^{\mathrm{sign}(A_i^{\mathrm{RL}})}
$$

然后：

$$
\hat A_{i,t}
=
A_i^{\mathrm{RL}}
\left[
(1-\lambda)
+
\lambda
\operatorname{clip}
\left(
w_{i,t}^{\mathrm{RLRT}},
1-\epsilon,
1+\epsilon
\right)
\right]
$$

这个组检验：

```text
teacher-student mismatch 在成功轨迹上到底是错误偏差，还是有价值的 self-driven reasoning。
```

如果 A5 有效，说明 OPD 不能只理解为“靠近 teacher”，还要区分 teacher mismatch 的符号和 rollout correctness。

**A6: privileged teacher 只调力度**

适用于 OPSD / PI-distillation 场景。设 privileged teacher evidence ratio：

$$
\Delta_{i,t}
=
\log \pi_T(a_{i,t}\mid s_{i,t},r)
-
\log \pi_S(a_{i,t}\mid s_{i,t})
$$

只把它用于 magnitude：

$$
w_{i,t}
=
\exp
\left(
\mathrm{sign}(A_i^{\mathrm{RL}})
\Delta_{i,t}
\right)
$$

$$
\hat A_{i,t}
=
A_i^{\mathrm{RL}}
\operatorname{clip}
\left(
w_{i,t},1-\epsilon,1+\epsilon
\right)
$$

这个组检验：

```text
在信息不对称下，OPD teacher 是否只能安全地调力度，而不能决定方向。
```

## 实验矩阵建议

建议最小实验矩阵如下：

| 组别 | 方向 | 力度 | 预期能回答的问题 |
|---|---|---|---|
| GRPO | sampled token | outcome advantage | RLVR baseline |
| sampled OPD | sampled token | teacher log-ratio | teacher 力度是否足够 |
| GRPO + OPD magnitude | sampled token | outcome advantage × teacher ratio | OPD 收益是否来自更细 credit |
| full/top-k OPD | distribution gap | KL/probability gap | teacher 方向是否关键 |
| OPD direction + RL advantage | distribution gap | outcome advantage | teacher direction 是否需要 RL correctness gate |
| SCOPE-style route | correct 用 self/RL，wrong 用 teacher | student/teacher PPL | 正误轨迹是否需要不同组合 |
| RLSD-style | RLVR direction | PI teacher magnitude | PI 场景是否必须方向/力度解耦 |
| RLRT-style | correct sampled token | reversed teacher ratio | 成功轨迹上的 teacher mismatch 是否有价值 |

## 评价指标

不能只看最终 accuracy。建议同时看：

**1. 任务指标**

- pass@1 / pass@k；
- average@k；
- verifier reward；
- code pass rate；
- agentic success rate。

**2. 训练动态**

- policy entropy；
- KL to teacher；
- KL to old policy；
- gradient norm；
- advantage / token weight 分布；
- positive / negative update ratio；
- update cosine similarity。

**3. 方向 vs 力度诊断**

对每个 batch 记录：

$$
\cos
\left(
g^{\mathrm{RL}},
g^{\mathrm{OPD}}
\right)
$$

其中：

- `g^RL` 是 GRPO gradient；
- `g^OPD` 是 OPD gradient。

如果 cosine 很高但 OPD 更好，说明两者方向类似，OPD 可能主要靠力度/credit assignment；
如果 cosine 很低且 OPD 更好，说明 teacher direction 提供了新方向。

也可以记录：

$$
\mathrm{Var}(c_{i,t})
$$

以及 token weight 与 correctness / teacher entropy / student entropy / divergence 的相关性。

**4. 分组指标**

必须按这些组拆：

- correct rollout vs wrong rollout；
- high teacher entropy vs low teacher entropy；
- high student entropy vs low student entropy；
- high teacher-student divergence vs low divergence；
- early reasoning token vs late verification token；
- sampled token teacher-supported vs teacher-disfavored。

否则容易把 OPD 的 compression、correction、imitation、exploration 混在一起。

## 可能结论模式

**模式 1：A1 接近 A3，A2 不强**

说明 OPD 的主要收益来自力度：

```text
teacher 不一定提供新方向；
它主要把稀疏 trajectory reward 细化成 token-level credit。
```

这支持 RLSD、sampled-token OPD、REOPOLD 这类方向不变 / 力度调制方法。

**模式 2：A2 / full-top-k OPD 明显强于 A1**

说明 teacher direction 重要：

```text
student 没采到的 token / branch 很关键；
只在 sampled token 上调力度不够。
```

这支持 full-vocab / top-k FKL、AOPD non-positive branch、TIP Q3 token selection 等方法。

**模式 3：correct rollout 上 A5 强，wrong rollout 上 A2/A4 强**

说明 OPD 需要 correctness routing：

```text
正确轨迹:
  teacher 不能压制 student self-driven path

错误轨迹:
  teacher direction / correction 更重要
```

这支持 SCOPE + RLRT 的组合思路。

**模式 4：PI 场景中 A6 强，full OPSD 不稳定**

说明 privileged teacher 不能决定方向，只能调力度：

```text
直接 matching privileged teacher distribution 会 leakage；
RLVR/verifier outcome 应该 anchor update direction。
```

这支持 RLSD 的机制归因。

**模式 5：所有 OPD 组合都不如 GRPO**

可能说明：

- teacher 没有 novelty；
- teacher-student thinking pattern 不兼容；
- teacher 在 flawed prefix 上不可靠；
- OPD 主要在压缩而不是纠错；
- 任务需要探索 teacher 不知道的新策略。

这与 Rethinking OPD 和 OPSD Compresses 的结论一致。

## 一个可写成论文的问题表述

可以把研究问题写成：

```text
Does OPD improve LLM post-training by changing the direction of policy updates,
or by providing better token-level credit magnitude?
```

中文表述：

```text
OPD 的收益到底来自 teacher 提供了新的更新方向，
还是来自 teacher 对 RLVR 稀疏 reward 做了更细粒度的 token-level 力度校准？
```

核心实验就是 direction-magnitude factorial design：

$$
\text{Update}
=
\text{Direction Source}
\times
\text{Magnitude Source}
$$

其中：

```text
Direction Source:
  RLVR sampled-token direction
  OPD sampled-token direction
  OPD full/top-k distribution direction
  reversed teacher direction / self-driven direction

Magnitude Source:
  outcome advantage
  teacher-student log-ratio
  teacher confidence / PPL
  student confidence / PPL
  privileged evidence ratio
  learned router weight
```

## 初步判断

基于现有论文，最可能的答案不是单一的：

```text
OPD 的收益在简单 strong-to-weak distillation 中，更多来自 teacher direction + dense signal；
在 RLVR sparse reward 场景中，更多来自 token-level magnitude / credit assignment；
在 PI self-distillation 中，teacher direction 可能有害，teacher 只适合调 magnitude；
在 correct rollouts 上，teacher direction 可能压制 student，有时需要 reversed teacher signal；
在 wrong rollouts 或 non-positive advantage 区域，teacher direction / top-k FKL 更可能提供纠错。
```

因此最有价值的方向不是证明“OPD 比 RLVR 好”，而是建立一个 router：

```text
给定一个 token / prefix / trajectory，
判断应该使用：

1. RLVR direction + RLVR magnitude
2. RLVR direction + OPD magnitude
3. OPD direction + RLVR magnitude
4. OPD direction + OPD magnitude
5. reversed teacher magnitude
6. skip / mask
```

这可以把 SCOPE、AOPD、RLSD、RLRT、TIP、EOPD 放进一个统一框架：它们都在回答同一个问题，即 **在哪些 token 或 trajectory 上，teacher 应该决定方向，哪些地方 teacher 只应该决定力度，哪些地方不该用 teacher**。
